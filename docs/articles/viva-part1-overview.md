ITSM/ITOM/ITAM 시리즈 1편 — 도대체 뭐가 다른 건가요?

요즘 ITSM, ITOM, ITAM 용어가 주변에서 자주 보이지 않으십니까? 사람마다 이해하는 범위가 달라서 회의 때 미묘하게 어긋나는 경험, 저만 있었던 건 아닐 겁니다.

찾아보니 레스토랑 비유가 가장 쉬웠습니다.

- **ITSM** = 홀 서비스. 직원이 "노트북 고장 났어요"라고 요청하면 접수하고 해결해주는 체계입니다.
- **ITOM** = 주방 운영. 서버, 네트워크 같은 인프라가 안 멈추도록 24시간 모니터링하고 사전에 잡아내는 역할입니다.
- **ITAM** = 재고 관리. 노트북이 몇 대 있고, 라이선스는 몇 개 남았고, 보증은 언제까지인지 추적하는 겁니다.

우리 회사 상황에 대입하면 이렇습니다. 금요일 오후 고객센터 시스템이 느려지면 — ITOM이 "빌링 서버 CPU 95% 초과"를 먼저 감지하고, ITSM이 장애 티켓을 자동 생성해서 담당자에게 배정하고, ITAM이 해당 서버의 도입 시기와 보증 상태를 알려줍니다.

결국 **ITOM이 감지하고, ITSM이 처리하고, ITAM이 자산 판단을 돕는** 구조이거든요.

그리고 이걸 체계적으로 하기 위한 가이드가 ITIL 4(모범 사례집), COBIT(거버넌스), ISO 20000(국제 인증)입니다.

다음 편에서는 ITSM을 좀 더 깊이 들여다봅니다 — 서비스 데스크, 인시던트 관리, SLA 이야기입니다.
