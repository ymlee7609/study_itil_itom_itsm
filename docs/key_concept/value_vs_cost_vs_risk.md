# Value vs Cost vs Risk

ITIL 4 **가치 공식의 3요소**. 의사결정 시 세 항목을 분리하지 못하면 "비용 절감"이 가치 파괴로 이어지거나, "리스크 방지"가 기회 상실로 이어진다.

## 1. 공식 정의

| 구분 | Value (가치) | Cost (비용) | Risk (리스크) |
|------|-------------|------------|---------------|
| **정의** | 서비스의 **인지된 이점·유용성·중요성** | 서비스 제공·소비에 **투입되는 자원** | 불확실성으로 발생 가능한 **손실 또는 기회** |
| **부호** | (+) 이익 | (−) 자원 소모 | (±) 양방향 (부정적 위협 + 긍정적 기회) |
| **시간 축** | 사용 중 실현 | 사전 투입 | 미래 불확실성 |
| **측정** | 정성+정량 (만족도, 매출) | 주로 정량 (금액) | 확률 × 영향도 |
| **담당 관점** | 소비자 | 제공자+소비자 | 양쪽 모두 |

---

## 2. 가치 공식

ITIL 4의 정식 정의:

```
Value = f(Utility, Warranty, Cost, Risk)
     = (Utility × Warranty) − Cost ± Risk
```

**고객 관점의 의사결정**:
- Utility(유용성) + Warranty(보증성)이 **크면** → 서비스 수용
- Cost(비용)가 **작으면** → 수용 매력 증가
- Risk가 **낮으면** → 수용 확신 증가

서비스 제공자의 임무: **가치 ↑ + 비용 ↓ + 리스크 ↓** 동시 달성.

### Cost의 두 얼굴

| 구분 | Cost Removed (제거되는 비용) | Cost Imposed (부과되는 비용) |
|------|-----------------------------|------------------------------|
| 의미 | 서비스 사용으로 **고객이 절감**하는 비용 | 서비스 구매·운영에 **새로 드는** 비용 |
| 예 | 자체 메일 서버 구축 불필요 | 클라우드 구독료 |
| 가치에 미치는 영향 | (+) 가치 증가 | (−) 가치 감소 |

### Risk의 두 얼굴

| 구분 | Risk Removed (제거되는 리스크) | Risk Imposed (부과되는 리스크) |
|------|-------------------------------|---------------------------------|
| 의미 | 서비스 사용으로 **고객이 회피**하는 리스크 | 서비스 사용으로 **새로 생기는** 리스크 |
| 예 | 서버 운영·보안 사고 부담 이전 | 벤더 락인, 서비스 중단 의존 |

## 3. 실무 예시

### 예시 1: 사내 이메일 → SaaS (Microsoft 365) 전환

| 항목 | 내용 |
|------|------|
| **Value** | 안정적 메일/협업, 모바일 접근, 통합 인증, 자동 업데이트 |
| **Cost Removed** | 온프레 서버 운영비(연 2억), 관리자 1명 인건비(연 1억), 백업·DR 비용 |
| **Cost Imposed** | M365 라이선스 연 1.5억, 사용자 교육비, 마이그레이션 비용 |
| **Risk Removed** | 서버 장애, 데이터 유실, 보안 패치 지연 |
| **Risk Imposed** | 벤더 락인, 인터넷 장애 시 전체 마비, 데이터 국외 이전 이슈 |
| **순 가치** | Value 크고, 순 비용 ↓ 1.5억/년, 주요 리스크 제거 → **도입 타당** |

### 예시 2: 결제 이중화 투자

| 항목 | 내용 |
|------|------|
| **Value** | 고객 이탈 방지, 평판 유지 |
| **Cost** | 이중 PG사 계약 연 3천만원, 운영 복잡도 증가 |
| **Risk (미투자 시)** | 단일 PG 장애 시 시간당 매출 손실 5천만원 × 연 2회 예상 = **1억/년** |
| **결론** | Risk 회피 이익(1억) > Cost(3천) → **투자 타당** |

### 예시 3: 서비스 정의에서의 Cost/Risk

> "서비스는 고객이 원하는 outcome 달성을 촉진하는 수단이다. **고객이 특정 비용과 리스크를 부담하지 않도록** 하면서." — ITIL 4

→ 서비스의 본질은 **Cost와 Risk를 대신 짊어지는 것**.

## 4. 왜 이 구분이 중요한가

### (1) "비용 절감"만 보면 Value 파괴

- 비용 절감 → 품질 저하 → Value 감소
- 비용 절감 시 반드시 **Value와 Risk 변동**도 함께 측정
- 예: 콜센터 외주로 30% 절감 → 고객 만족도 20% 하락 → 이탈 증가

### (2) "리스크 제로"는 기회 상실

- 모든 리스크를 회피하려 하면 **변화 거부** → 경쟁력 상실
- ITIL은 Risk를 **관리 대상**이지 **제거 대상**이 아니라고 정의
- 리스크 수용(Accept), 이전(Transfer), 완화(Mitigate), 회피(Avoid) 4개 옵션 중 선택

### (3) Value는 고객이 느끼는 것

- 제공자가 "가치 있다" 해도 고객이 느끼지 못하면 Value = 0
- **체감 가치** 측정 필수 (설문, NPS, 사용률)
- → XLA 측정의 근거

### (4) 트레이드오프 명시화

의사결정 시 3개 축을 별도 컬럼으로:

| 옵션 | Value | Cost | Risk | 결론 |
|------|-------|------|------|------|
| A. 현상 유지 | 중 | 중 | 고 | 기각 |
| B. 클라우드 전환 | 고 | 고 | 중 | 채택 |
| C. 부분 개선 | 저 | 저 | 고 | 보류 |

### (5) ITIL 5 관점

ITIL 5의 AI 거버넌스 **6C 모델**은 Value·Cost·Risk 프레임을 확장:
- Confidence (신뢰) = Risk 관리
- Continuity (연속성) = Warranty 보강
- 경험(XLA) = Value 체감 측정

## 5. 한눈 정리

```
                     Value (가치)
                         ▲
                         │
                    소비자가 느낌
                         │
   ─────────────────────┼─────────────────────
   Cost (비용)           │           Risk (리스크)
   ▼                     │                     ▼
자원 투입               트레이드오프          불확실성
Removed / Imposed                           Removed / Imposed
사전 계산               균형 필수           확률 × 영향
```

---

## 한 줄 요약

**Value는 고객이 느끼는 이점, Cost는 투입되는 자원, Risk는 불확실성에 따른 손실·기회.** 셋은 트레이드오프 관계이며, Cost와 Risk는 "제거"와 "부과"의 양면을 갖는다. 서비스의 본질은 고객의 Cost와 Risk를 대신 짊어지는 것.
