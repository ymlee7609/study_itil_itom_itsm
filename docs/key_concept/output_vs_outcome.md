# Output vs Outcome

ITIL 4의 **서비스 가치 공식**을 이해하는 핵심 개념. 둘은 자주 혼용되지만 **완전히 다른 층위**에 있다.

## 1. 공식 정의

| 구분 | Output (산출물) | Outcome (결과) |
|------|----------------|----------------|
| **정의** | 활동의 **유형적·무형적 결과물** | 이해관계자가 **달성하고자 하는 결과나 변화** |
| **주체 관점** | **서비스 제공자**가 만드는 것 | **서비스 소비자**가 얻는 것 |
| **측정 대상** | "무엇을 만들었나" (행위/생산) | "어떤 변화가 일어났나" (가치/효과) |
| **통제 가능성** | 제공자가 직접 통제 가능 | 제공자 단독 통제 불가 (소비자 환경 의존) |
| **영어 어원** | put out (내놓다) | come out (튀어나온 것, 귀결) |

---

## 2. 관계식

```
 Output    +    사용자 활동/맥락   =    Outcome
(산출물)        (소비자 자원)         (결과·가치)
```

ITIL 4는 이를 **"서비스는 output을 제공하지만, 가치는 outcome에서 발생한다"**고 명시한다.

---

## 3. 실무 예시

### 예시 1: 이메일 시스템

| 항목 | 내용 |
|------|------|
| **Output** | 99.9% 가용한 메일 서버, 5GB 메일함, 스팸 필터 |
| **Outcome** | 직원 간 원활한 커뮤니케이션, 고객 응대 속도 향상, 업무 생산성 증가 |

→ 서버가 돌아가도(Output 달성) 직원이 메일을 안 쓰면 Outcome은 0.

### 예시 2: 인시던트 관리

| 항목 | 내용 |
|------|------|
| **Output** | 인시던트 티켓 1,200건 처리, MTTR 2시간 |
| **Outcome** | 영업팀이 중단 없이 계약 체결, 매출 손실 방지 |

### 예시 3: 신규 CRM 도입

| 항목 | 내용 |
|------|------|
| **Output** | CRM 시스템 구축, 사용자 교육 완료, 매뉴얼 배포 |
| **Outcome** | 영업 전환율 15% 상승, 고객 재구매율 증가, 영업 사이클 단축 |

## 4. 왜 이 구분이 중요한가

### (1) SLA vs XLA 차이의 뿌리

- **SLA (Service Level Agreement)** = Output 중심 (가동률, 응답시간 등 제공자 측정치)
- **XLA (Experience Level Agreement)** = Outcome 중심 (사용자 체감·업무 성과) ← ITIL 5에서 공식 강조

### (2) "워터멜론 효과" 방지

SLA는 전부 녹색(Output 달성)인데 사용자 만족도는 빨갛게 낮은 현상. Output만 쳐다보면 Outcome을 놓친다는 경고다.

### (3) 서비스의 정의 자체가 Outcome 기반

> "서비스란 **고객이 원하는 outcome의 달성을 촉진**하는 수단이다. 고객이 **특정 비용과 리스크를 부담하지 않도록** 하면서." — ITIL 4

서비스 제공자의 존재 이유가 "Output 생산"이 아니라 "고객의 Outcome 촉진"이라는 철학이 담긴 정의다.

---

## 5. 한눈 정리

```
제공자 측              │              소비자 측
─────────────────────┼─────────────────────
 활동 (Activity)       │
   │                  │
   ▼                  │
 산출물 (Output) ───► 사용 ───► 결과 (Outcome) ───► 가치 (Value)
 "무엇을 제공했나"      │  "어떤 변화를 얻었나"      "얼마나 이득인가"
```

---

## 6. ITIL 5 관점에서의 강화

ITIL 5 (2026.2)는 **경험 관리(Experience Management)**를 공식화하면서 Outcome을 더 강조한다:

- Output 지표 (SLA) → Outcome 지표 (XLA)로 측정 무게중심 이동
- 제품·서비스 라이프사이클(PSLM) 전체가 **고객 Outcome 실현**을 축으로 구성
- AI 거버넌스 6C 모델의 **Confidence(신뢰)·Continuity(연속성)**도 Outcome 보호 관점

---

## 한 줄 요약

Output은 "내가 무엇을 만들었나(제공자 시점)", Outcome은 "고객에게 어떤 변화가 일어났나(소비자 시점)". ITIL은 Output 생산이 아닌 **Outcome 촉진**을 서비스의 본질로 정의한다.
