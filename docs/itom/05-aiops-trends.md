## 1. AIOps 정의

**AIOps (AI for IT Operations)**는 인공지능과 머신러닝을 활용하여 IT 운영을 자동화하고 최적화하는 접근 방식이다. ITSM과 ITOM을 연결하는 **통합 플랫폼** 역할을 수행한다.

| 항목 | 설명 |
|------|------|
| 정의 | AI/ML을 활용한 IT 운영 자동화 및 최적화 플랫폼 |
| 기원 | 2017년 Gartner가 처음 정의 |
| 목적 | IT 운영의 복잡성 관리, 사전 예방적 운영, 자동화 |
| 위치 | [ITOM](./01-overview.md)과 [ITSM](../itsm/) 사이의 지능형 계층 |
| 핵심 가치 | 노이즈 감소, 근본 원인 자동 분석, 예측 기반 운영 |

### AIOps의 진화

| 세대 | 시기 | 특징 | 기술 |
|------|------|------|------|
| 1세대 | 2017-2019 | 규칙 기반 자동화 | 정적 임계값, 기본 상관분석 |
| 2세대 | 2019-2022 | ML 기반 분석 | 이상 탐지, 패턴 인식, 클러스터링 |
| 3세대 | 2022-2024 | 인과관계 AI | 근본 원인 추론, 예측 분석 |
| 4세대 | 2024-현재 | 생성형 AI 통합 | LLM 기반 분석, 자율 복구, 자연어 쿼리 |

## 2. AIOps 핵심 기능

### 기능 개요

| 기능 | 설명 | 효과 |
|------|------|------|
| 노이즈 감소 (Noise Reduction) | 중복/관련 알림 자동 그룹핑 및 필터링 | 알림 90%+ 감소 |
| 이벤트 상관분석 (Event Correlation) | 시간/공간/인과관계 기반 이벤트 연결 | 문제 파악 시간 단축 |
| 이상 탐지 (Anomaly Detection) | ML 기반 비정상 패턴 자동 식별 | 장애 사전 예방 |
| 근본 원인 분석 (Root Cause Analysis) | AI 기반 장애 원인 자동 추론 | MTTR 60%+ 단축 |
| 예측 분석 (Predictive Analytics) | 장애/용량/성능 문제 사전 예측 | 사전 대응 가능 |
| 자동 인시던트 생성 | 분석 결과 기반 [인시던트](../itsm/) 자동 생성 | 대응 시간 단축 |
| 지능형 라우팅 (Intelligent Routing) | 인시던트를 최적 담당자/팀에 자동 배정 | 해결 시간 단축 |

### 노이즈 감소 상세

| 기법 | 설명 | 감소율 |
|------|------|--------|
| 중복 제거 (Deduplication) | 동일 알림 그룹핑 | 30-40% |
| 상관분석 (Correlation) | 관련 알림 연결 | 20-30% |
| 억제 (Suppression) | 유지보수 중 알림 억제 | 10-20% |
| 적응형 임계값 (Adaptive Thresholds) | ML 기반 동적 임계값 | 10-15% |
| 총 감소 효과 | - | **90%+** |

### 이상 탐지 기법

| 기법 | 설명 | 적용 영역 |
|------|------|-----------|
| 통계적 방법 (Statistical) | 평균/표준편차 기반 이상 판단 | 메트릭 모니터링 |
| 시계열 분석 (Time Series) | 계절성/트렌드 기반 예측 및 이탈 탐지 | 용량 계획, 성능 모니터링 |
| 클러스터링 (Clustering) | 정상 패턴 그룹화, 이상 패턴 분리 | 로그 분석, 행동 분석 |
| 딥러닝 (Deep Learning) | LSTM/Transformer 기반 복잡한 패턴 학습 | 복합 이상 탐지 |
| 그래프 분석 (Graph Analysis) | 토폴로지 기반 전파 패턴 분석 | 서비스 맵 기반 분석 |

## 4. 2025-2026 주요 트렌드

### 트렌드 개요

| 번호 | 트렌드 | 영향도 | 성숙도 |
|------|--------|--------|--------|
| 1 | ITSM + ITOM 통합 | 매우 높음 | 성숙 |
| 2 | AIOps 주도 자동화 | 매우 높음 | 성장 |
| 3 | 멀티클라우드/하이브리드 환경 지원 | 높음 | 성숙 |
| 4 | 제로 트러스트 보안 (Zero Trust Security) | 높음 | 성장 |
| 5 | AI/ML 기반 예측 운영 | 높음 | 성장 |
| 6 | DEX (Digital Employee Experience) | 중간 | 초기 |
| 7 | CI/CD 파이프라인 모니터링 | 높음 | 성숙 |
| 8 | Observability 2.0과 OpenTelemetry | 매우 높음 | 성장 |

### 1. ITSM + ITOM 통합

| 항목 | 설명 |
|------|------|
| 동인 | 사일로 제거, 엔드투엔드 가시성 필요 |
| 방향 | 단일 플랫폼에서 운영+서비스 관리 통합 |
| 사례 | ServiceNow (ITOM+ITSM), BMC Helix |
| 참조 | [ITOM vs ITSM 비교](./02-itom-vs-itsm.md) |

### 2. AIOps 주도 자동화

| 항목 | 설명 |
|------|------|
| 동인 | IT 복잡성 증가, 인력 부족 |
| 방향 | 감지 → 분석 → 조치까지 자동 파이프라인 |
| 핵심 기술 | 생성형 AI, 강화학습, 자율 복구 |
| 사례 | 자동 장애 복구, 지능형 변경 관리 |

### 3. 멀티클라우드/하이브리드 환경 지원

| 항목 | 설명 |
|------|------|
| 동인 | 기업의 멀티클라우드 전략 확대 |
| 과제 | 통합 가시성, 일관된 정책, 비용 관리 |
| 핵심 기술 | 클라우드 추상화 계층, FinOps |
| 도구 | ServiceNow Cloud Mgmt, Datadog, CloudHealth |

### 4. 제로 트러스트 보안 (Zero Trust Security)

| 항목 | 설명 |
|------|------|
| 동인 | 원격근무 확대, 클라우드 전환, 위협 증가 |
| ITOM 영향 | 네트워크 세그먼테이션, 지속적 인증, 마이크로세그먼테이션 모니터링 |
| 핵심 원칙 | "절대 신뢰하지 않고, 항상 검증" |
| 연계 | 보안 이벤트 모니터링, 접근 제어 자동화 |

### 5. AI/ML 기반 예측 운영

| 항목 | 설명 |
|------|------|
| 동인 | 사후 대응에서 사전 예방으로 전환 |
| 적용 영역 | 장애 예측, 용량 예측, 성능 이상 예측 |
| 핵심 기술 | 시계열 예측, 딥러닝, 자연어 처리 |
| 효과 | 계획되지 않은 장애 60-80% 감소 |

### 6. DEX (Digital Employee Experience)

| 항목 | 설명 |
|------|------|
| 정의 | 직원의 디지털 업무 환경 경험 관리 |
| ITOM 연계 | 엔드포인트 성능 모니터링, 앱 응답시간 측정 |
| 도구 | Nexthink, Lakeside, Aternity |
| 측정 지표 | DEX 점수, 앱 성능, 디바이스 건강도 |

### 7. CI/CD 파이프라인 모니터링

| 항목 | 설명 |
|------|------|
| 동인 | DevOps 확대, 배포 빈도 증가 |
| ITOM 연계 | 배포 영향 분석, 자동 롤백, 카나리 모니터링 |
| 도구 | Datadog CI Visibility, Dynatrace, ArgoCD |
| 핵심 가치 | 배포 실패 조기 감지, 자동 롤백 |

### 8. Observability 2.0과 OpenTelemetry

| 항목 | 설명 |
|------|------|
| 정의 | 벤더 중립적 통합 관측성 표준 |
| OpenTelemetry (OTel) | CNCF 프로젝트, 메트릭/로그/트레이스 통합 수집 |
| 장점 | 벤더 종속 탈피, 표준화된 계측, 유연한 백엔드 선택 |
| 채택 현황 | Datadog, New Relic, Dynatrace, Grafana 등 주요 벤더 지원 |

**Observability 1.0 vs 2.0:**

| 비교 | Observability 1.0 | Observability 2.0 |
|------|-------------------|-------------------|
| 데이터 | 메트릭, 로그, 트레이스 분리 | 통합 텔레메트리 데이터 |
| 표준 | 벤더별 독자 형식 | OpenTelemetry 표준 |
| 분석 | 대시보드 기반 수동 분석 | AI 기반 자동 분석 |
| 접근 | 사후 분석 (Reactive) | 사전 예방 (Proactive) |
| 상관관계 | 수동 연결 | 자동 상관분석 |

## 5. 자율 운영의 미래

### 자율 운영 성숙도 (Autonomous Operations Maturity)

| 수준 | 단계 | 설명 | 인간 개입 |
|------|------|------|-----------|
| 0 | 수동 (Manual) | 모든 작업을 사람이 수행 | 100% |
| 1 | 보조 (Assisted) | 도구가 정보 제공, 사람이 판단 | 80% |
| 2 | 부분 자동화 (Partial) | 일부 작업 자동화, 사람이 감독 | 50% |
| 3 | 조건부 자동화 (Conditional) | 대부분 자동, 예외 시 사람 개입 | 20% |
| 4 | 고도 자동화 (High) | 거의 모든 작업 자동, 정책 기반 | 5% |
| 5 | 완전 자율 (Full) | AI가 판단하고 실행, 사람은 전략만 | 1% |

### 자율 운영 핵심 기술

| 기술 | 역할 | 현재 성숙도 |
|------|------|-----------|
| 생성형 AI (GenAI) | 자연어 쿼리, 런북 자동 생성, 장애 요약 | 초기 |
| 강화학습 (RL) | 최적 조치 학습, 자동 튜닝 | 실험 |
| 인과관계 추론 (Causal AI) | 근본 원인 자동 식별 | 성장 |
| 디지털 트윈 (Digital Twin) | IT 환경 시뮬레이션, What-if 분석 | 초기 |
| 자율 복구 (Self-Healing) | 장애 자동 감지 및 복구 | 성장 |

### 생성형 AI의 ITOM 적용

| 적용 영역 | 설명 | 사례 |
|-----------|------|------|
| 자연어 쿼리 | "지난 1시간 동안 지연이 높은 서비스는?" | Datadog Bits AI, ServiceNow Now Assist |
| 장애 요약 | 인시던트 컨텍스트 자동 요약 | PagerDuty Copilot |
| 런북 생성 | 장애 대응 절차 자동 작성 | ServiceNow Now Assist |
| 코드 분석 | 배포 변경이 장애 원인인지 분석 | Dynatrace Davis CoPilot |
| 보고서 생성 | 운영 보고서 자동 생성 | 각 플랫폼 AI 기능 |

## 6. AIOps 도입 로드맵

### 단계별 도입

| 단계 | 기간 | 활동 | 목표 |
|------|------|------|------|
| 1. 기반 구축 | 0-3개월 | 모니터링 통합, 데이터 수집 표준화 | 통합 데이터 파이프라인 |
| 2. 노이즈 감소 | 3-6개월 | 이벤트 상관분석, 중복 제거 | 알림 80% 감소 |
| 3. 이상 탐지 | 6-12개월 | ML 모델 학습, 동적 임계값 | 사전 장애 감지 |
| 4. 자동 분석 | 12-18개월 | 근본 원인 자동 분석, 영향 분석 | MTTR 50% 단축 |
| 5. 자동 복구 | 18-24개월 | 자동 조치, 자율 복구 | 자동 해결률 30%+ |
| 6. 예측 운영 | 24개월+ | 예측 분석, 자율 운영 | 장애 사전 예방 |

### 성공 요인

| 요인 | 설명 |
|------|------|
| 데이터 품질 | 정확하고 일관된 모니터링 데이터 확보 |
| 점진적 접근 | 한 번에 모든 것을 하지 않고 단계적 도입 |
| 조직 변화 관리 | 운영팀의 역할 변화에 대한 교육과 지원 |
| 경영진 지원 | 투자 정당성 확보, ROI 측정 |
| 도메인 전문성 | AI/ML 기술 + IT 운영 도메인 지식 결합 |
| [ITIL 4 프레임워크](../itil4/01-overview.md) 연계 | 프로세스 표준화 기반 위에 AI 적용 |

### KPI 측정 프레임워크

| KPI | 도입 전 | 목표 (12개월) | 목표 (24개월) |
|-----|---------|-------------|-------------|
| 알림 노이즈 감소율 | 0% | 80% | 95% |
| MTTR (평균 복구 시간) | 4시간 | 2시간 | 30분 |
| 자동 인시던트 생성률 | 0% | 50% | 80% |
| 사전 장애 감지율 | 10% | 40% | 70% |
| 자동 복구 비율 | 0% | 15% | 30% |
