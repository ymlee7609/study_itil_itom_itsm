## 1. ITSM 정의

**ITSM (IT Service Management)**은 IT 팀이 고객에게 IT 서비스를 **설계, 생성, 전달, 지원**하는 end-to-end 프로세스와 활동을 관리하는 **전략적 접근 방식**이다.

> ITSM의 핵심 관점: IT를 단순한 기술 운영이 아닌 **비즈니스 가치를 전달하는 서비스**로 바라본다.

| 구분 | 설명 |
|------|------|
| 정의 | IT 서비스의 전체 수명주기를 관리하는 체계적 접근법 |
| 목적 | IT 서비스 품질 향상 및 비즈니스 목표 달성 지원 |
| 범위 | 서비스 전략, 설계, 전환, 운영, 지속적 개선 |
| 대상 | IT 조직, 서비스 제공자, 최종 사용자 |
| 핵심 원칙 | 서비스 중심 사고, 프로세스 기반 관리, 지속적 개선 |

### ITSM의 역사적 발전

| 시기 | 발전 단계 | 주요 특징 |
|------|-----------|-----------|
| 1980년대 | 초기 형성기 | 영국 정부(CCTA)가 IT 서비스 관리 모범 사례 수집 시작 |
| 1990년대 | ITIL v1/v2 | 체계적인 프레임워크로 발전, 프로세스 중심 |
| 2000년대 | ITIL v3 | 서비스 수명주기(Service Lifecycle) 개념 도입 |
| 2010년대 | 클라우드/DevOps 통합 | 애자일, DevOps와의 융합 시작 |
| 2019년~ | ITIL 4 | 가치 공동 창출(Value Co-creation), SVS 도입 |

## 2. 핵심 철학

### IT는 서비스로서 작동한다

ITSM은 IT 부서를 **내부 서비스 제공자(Internal Service Provider)**로 정의한다. 기술 자체가 아닌, 기술이 전달하는 **가치(Value)**에 초점을 맞춘다.

| 전통적 IT 관점 | ITSM 관점 |
|----------------|-----------|
| 기술 중심 (Technology-centric) | 서비스 중심 (Service-centric) |
| 인프라 관리에 집중 | 비즈니스 가치 전달에 집중 |
| 사후 대응 (Reactive) | 사전 예방 (Proactive) |
| IT 부서 내부 시각 | 최종 사용자 경험 시각 |
| 기술적 SLA 중심 | 비즈니스 성과 중심 |
| 개별 시스템 관리 | End-to-End 서비스 관리 |

### 최종 사용자 중심 (End-User Centric)

- **서비스 경험(Service Experience)**: 사용자가 IT 서비스를 이용하는 전체 경험을 관리
- **셀프 서비스(Self-Service)**: 사용자가 직접 요청하고 해결할 수 있는 채널 제공
- **투명성(Transparency)**: 서비스 상태, 요청 진행 상황을 사용자에게 공개

## 3. 핵심 구성요소

ITSM은 **세 가지 핵심 축**으로 구성된다.

| 구성요소 | 영문 | 설명 | 예시 |
|----------|------|------|------|
| 사람 | People | 역할, 책임, 역량, 문화 | 서비스 데스크 분석가, 변경 관리자, 프로세스 오너 |
| 프로세스 | Process | 정의된 절차, 워크플로우, 정책 | 인시던트 관리, 변경 관리, 문제 관리 프로세스 |
| 기술 | Technology | 도구, 자동화, 플랫폼 | ITSM 도구(ServiceNow, Jira SM), 모니터링 시스템 |

### 구성요소 간 상호작용

```
        사람 (People)
       /            \
      /              \
프로세스 (Process) --- 기술 (Technology)
```

- **사람 + 프로세스**: 역할과 책임이 정의된 표준 절차 수행
- **프로세스 + 기술**: 자동화된 워크플로우를 통한 효율적 실행
- **기술 + 사람**: 도구를 활용한 생산성 향상과 의사결정 지원

> 세 요소 중 하나라도 부족하면 ITSM은 효과적으로 작동하지 않는다. 특히 **사람(People)** 요소는 가장 간과되기 쉽지만 성공의 핵심이다.

## 4. ITSM 프레임워크

ITSM을 구현하기 위한 다양한 프레임워크가 존재한다.

| 프레임워크 | 제공 조직 | 주요 특징 | 적합한 조직 |
|------------|-----------|-----------|-------------|
| ITIL (IT Infrastructure Library) | Axelos/PeopleCert | 가장 널리 채택된 ITSM 프레임워크, 서비스 가치 시스템(SVS) | 모든 규모의 조직 |
| COBIT (Control Objectives for Information and Related Technologies) | ISACA | IT 거버넌스와 관리 중심, 감사/규정 준수에 강점 | 규제 산업, 대기업 |
| ISO/IEC 20000 | ISO | ITSM 국제 표준, 인증 가능 | 인증이 필요한 서비스 제공자 |
| FitSM | ITEMO | 경량(Lightweight) ITSM 표준, 14개 프로세스 | 중소기업, 연구기관 |
| MOF (Microsoft Operations Framework) | Microsoft | Microsoft 기술 스택에 최적화 | Microsoft 환경 중심 조직 |
| VeriSM | IFDC | 서비스 관리 메쉬(Management Mesh) 접근 | 디지털 전환 추진 조직 |

### 프레임워크 선택 기준

| 기준 | ITIL | COBIT | ISO 20000 | FitSM |
|------|------|-------|-----------|-------|
| 구현 복잡도 | 중~높음 | 높음 | 중간 | 낮음 |
| 인증 체계 | 개인 인증 | 개인 인증 | 조직 인증 | 개인/조직 |
| 비용 | 중간 | 높음 | 높음 | 낮음 |
| 커뮤니티 규모 | 매우 큼 | 큼 | 중간 | 작음 |
| DevOps 호환성 | 높음 (ITIL 4) | 중간 | 중간 | 높음 |

## 5. ITSM이 조직에 제공하는 가치

### 비즈니스 가치 요약

| 가치 영역 | 설명 | 기대 효과 |
|-----------|------|-----------|
| 운영 효율성 (Operational Efficiency) | 표준화된 프로세스로 반복 작업 감소 | 운영 비용 20~30% 절감 |
| 비용 절감 (Cost Reduction) | 자원 최적화, 중복 제거, 자동화 | IT 예산의 효율적 배분 |
| 고객 만족 (Customer Satisfaction) | 일관된 서비스 품질, 신속한 대응 | CSAT 점수 향상 |
| 규정 준수 (Compliance) | 감사 추적, 표준 프로세스, 문서화 | 감사 대응 시간 단축 |
| 리스크 관리 (Risk Management) | 변경 관리, 영향 분석, 롤백 절차 | 장애 발생률 감소 |
| 서비스 가시성 (Service Visibility) | 대시보드, KPI, SLA 모니터링 | 의사결정 속도 향상 |
| 지식 관리 (Knowledge Management) | 경험과 솔루션의 체계적 축적 | 해결 시간 단축, 인력 의존도 감소 |

### 정량적 효과 예시

| 지표 | ITSM 도입 전 | ITSM 도입 후 | 개선율 |
|------|-------------|-------------|--------|
| 평균 인시던트 해결 시간 (MTTR) | 8시간 | 4시간 | 50% 감소 |
| 첫 번째 연락 해결율 (FCR) | 40% | 70% | 75% 향상 |
| 변경 성공률 | 75% | 95% | 27% 향상 |
| 서비스 가용성 | 98.5% | 99.9% | 향상 |
| 반복 인시던트 비율 | 30% | 10% | 67% 감소 |

## 6. ITSM 도입 전후 비교

| 영역 | 도입 전 (As-Is) | 도입 후 (To-Be) |
|------|-----------------|-----------------|
| 인시던트 처리 | 구두/이메일로 비체계적 처리 | 티켓 기반 체계적 관리 |
| 변경 관리 | 승인 없이 즉시 변경 | CAB 검토 및 승인 절차 |
| 지식 공유 | 개인의 암묵지에 의존 | Knowledge Base 활용 |
| 성과 측정 | 감에 의한 평가 | KPI/SLA 기반 정량 평가 |
| 서비스 요청 | 전화/메신저로 산발적 요청 | 셀프서비스 포털 활용 |
| 자산 관리 | 엑셀 기반 수동 관리 | CMDB 연동 자동화 |
