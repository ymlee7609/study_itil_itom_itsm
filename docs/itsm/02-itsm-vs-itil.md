## 1. 핵심 차이: "무엇" vs "어떻게"

| 구분 | ITSM | ITIL |
|------|------|------|
| 정의 | IT 서비스 관리라는 **분야/개념** | ITSM 구현을 위한 **프레임워크/지침** |
| 비유 | **요리(Cooking)** - 음식을 만드는 활동 전체 | **레시피(Recipe)** - 요리를 잘 하기 위한 방법론 |
| 성격 | What - 무엇을 해야 하는가 | How - 어떻게 해야 하는가 |
| 범위 | IT 서비스 관리의 모든 활동과 원칙 | ITSM 실천을 위한 구체적 가이드라인 |

### 관계 다이어그램

```
┌─────────────────────────────────────────────┐
│              ITSM (분야/개념)                  │
│                                             │
│   ┌──────┐  ┌────────┐  ┌───────────┐       │
│   │ ITIL │  │ COBIT  │  │ ISO 20000 │  ...  │
│   └──────┘  └────────┘  └───────────┘       │
│   (프레임워크들)                                │
└─────────────────────────────────────────────┘
```

> ITSM은 **상위 개념**이고, ITIL은 ITSM을 구현하기 위한 **여러 프레임워크 중 하나**이다. 가장 널리 채택되었기에 종종 동의어처럼 사용되지만, 엄밀히 다른 개념이다.

## 2. 상세 비교표

| 비교 항목 | ITSM | ITIL |
|-----------|------|------|
| **본질 (Essence)** | IT 서비스 관리라는 분야(discipline) | ITSM 모범 사례(best practice) 프레임워크 |
| **범위 (Scope)** | 모든 IT 서비스 관리 활동 포괄 | 34개 관리 실천(Management Practices) 정의 |
| **관계 (Relationship)** | 상위 개념 (umbrella term) | ITSM의 하위 구현 수단 |
| **적용 방식 (Application)** | 조직의 IT 서비스 관리 전략 수립 | 구체적인 프로세스, 역할, 가이드 제공 |
| **유연성 (Flexibility)** | 프레임워크에 구애받지 않음 | 구조화된 가이드라인 제공 (채택/적용 방식 유연) |
| **인증 (Certification)** | 특정 인증 없음 | ITIL Foundation, CDS, DPI, HVIT 등 |
| **소유권 (Ownership)** | 업계 공통 개념 | PeopleCert/Axelos 소유 |
| **발전 (Evolution)** | 기술 및 비즈니스 변화에 따라 자연 진화 | 버전별 명시적 업데이트 (v2 → v3 → 4) |
| **독립성** | 특정 도구/프레임워크 독립 | ITIL 관련 교육/인증 생태계 존재 |
| **구현 비용** | 프레임워크 선택에 따라 다양 | 교육/인증/컨설팅 비용 발생 |

## 3. ITIL 외 ITSM 프레임워크

### 주요 프레임워크 개요

| 프레임워크 | 초점 영역 | 제공 조직 | 주요 대상 |
|------------|-----------|-----------|-----------|
| **COBIT** (Control Objectives for Information and Related Technologies) | IT 거버넌스 및 관리 | ISACA | 경영진, 감사인, 규제 산업 |
| **ISO/IEC 20000** | ITSM 국제 표준, 인증 가능 | ISO/IEC | 서비스 제공 업체, 인증 필요 조직 |
| **MOF** (Microsoft Operations Framework) | Microsoft 환경 운영 관리 | Microsoft | Microsoft 기반 IT 조직 |
| **FitSM** | 경량(Lightweight) ITSM | ITEMO | 중소기업, 연구기관, 페더레이션 |
| **VeriSM** | 디지털 시대 서비스 관리 | IFDC | 디지털 전환 추진 조직 |
| **SIAM** (Service Integration and Management) | 멀티 공급자 관리 | Scopism | 여러 공급업체 관리 조직 |

## 4. 프레임워크별 상세 비교

### ITIL vs COBIT

| 비교 항목 | ITIL 4 | COBIT 2019 |
|-----------|--------|------------|
| 주요 초점 | 서비스 관리 실천(Practices) | IT 거버넌스와 관리(Governance) |
| 대상 사용자 | IT 운영팀, 서비스 관리자 | 경영진, 감사인, 이사회 |
| 프로세스 수 | 34개 관리 실천(Management Practices) | 40개 거버넌스/관리 목표 |
| 인증 대상 | 개인 | 개인 |
| 강점 | 서비스 운영/전달 상세 가이드 | 규정 준수, 리스크 관리, 감사 |
| 약점 | 거버넌스 측면 상대적 약함 | 운영 수준 상세 가이드 부족 |
| 상호보완 | COBIT으로 거버넌스 프레임 구축 | ITIL로 운영 프로세스 상세화 |

### ITIL vs ISO/IEC 20000

| 비교 항목 | ITIL 4 | ISO/IEC 20000-1:2018 |
|-----------|--------|----------------------|
| 성격 | 모범 사례 가이드(Best Practice) | 국제 표준(International Standard) |
| 인증 | 개인 인증 | **조직 인증** (제3자 감사) |
| 요구사항 | "~해야 한다(should)" 권고 | "~해야 한다(shall)" 의무 |
| 범위 | 포괄적 가이드라인 | SMS(Service Management System) 요구사항 |
| 비용 | 교육/인증 비용 | 감사/인증 유지 비용 추가 |
| 갱신 주기 | 비정기적 | ISO 표준 갱신 주기 (5년) |
| 활용 | 내부 개선 기준 | 외부 증명/계약 요건 충족 |

### ITIL vs FitSM

| 비교 항목 | ITIL 4 | FitSM |
|-----------|--------|-------|
| 규모 | 대규모, 상세 | 경량(Lightweight), 간결 |
| 프로세스 수 | 34개 관리 실천 | 14개 프로세스 |
| 학습 곡선 | 높음 (다수의 인증 레벨) | 낮음 (빠른 도입 가능) |
| 라이선스 | 상용 (PeopleCert) | 오픈소스 (Creative Commons) |
| 적합 조직 | 중대형 조직 | 중소기업, 학술/연구 기관 |
| 문서량 | 수천 페이지 | 수십 페이지 |

### MOF vs ITIL

| 비교 항목 | ITIL 4 | MOF 4.0 |
|-----------|--------|---------|
| 제공사 | PeopleCert/Axelos | Microsoft |
| 초점 | 범용 IT 서비스 관리 | Microsoft 기술 환경 운영 |
| 구조 | 서비스 가치 시스템(SVS) | SMF(Service Management Functions) |
| 기술 종속성 | 기술 중립 | Microsoft 생태계 최적화 |
| 비용 | 유료 교육/인증 | 무료 다운로드 |
| 현재 상태 | 활발한 업데이트 | 업데이트 중단 (레거시) |

## 5. 실무에서의 상호보완적 활용

### 일반적인 조합 패턴

| 조합 | 활용 목적 | 적합한 상황 |
|------|-----------|-------------|
| ITIL + COBIT | 서비스 관리 + 거버넌스 | 규제 산업(금융, 의료), 대기업 |
| ITIL + ISO 20000 | 모범 사례 + 인증 표준 | 서비스 제공업체, 아웃소싱 |
| ITIL + DevOps | 안정성 + 민첩성 | 디지털 전환 중인 조직 |
| ITIL + Agile | 서비스 관리 + 개발 방법론 | IT 조직 전체 최적화 |
| COBIT + ISO 20000 | 거버넌스 + 인증 | 강력한 컴플라이언스 요구 환경 |

### 단계적 도입 전략

| 단계 | 활동 | 권장 프레임워크 |
|------|------|-----------------|
| 1단계: 기반 구축 | 기본 프로세스 정립 (인시던트, 서비스 요청) | ITIL (Foundation 수준) 또는 FitSM |
| 2단계: 프로세스 성숙 | 변경 관리, 문제 관리, SLA 도입 | ITIL (중급 수준) |
| 3단계: 거버넌스 강화 | IT 거버넌스 체계 수립 | COBIT 추가 도입 |
| 4단계: 인증 획득 | 외부 검증, 공식 인증 | ISO/IEC 20000 인증 |
| 5단계: 통합 최적화 | DevOps, Agile 통합, 자동화 | ITIL 4 + DevOps/Agile |

## 6. 자주 묻는 질문 (FAQ)

| 질문 | 답변 |
|------|------|
| ITSM을 하려면 반드시 ITIL을 따라야 하나요? | 아니요. ITIL은 가장 널리 사용되는 프레임워크이지만, COBIT, FitSM 등 다른 프레임워크로도 ITSM을 구현할 수 있습니다. |
| ITIL 인증 없이 ITSM을 도입할 수 있나요? | 네. ITIL 인증은 개인의 지식 검증용이며, 조직의 ITSM 도입과 직접적 관련은 없습니다. |
| 소규모 조직에서는 어떤 프레임워크가 적합한가요? | FitSM이나 ITIL 4의 핵심 실천만 선별 채택하는 방식을 권장합니다. |
| ITIL 4와 DevOps는 충돌하나요? | 아니요. ITIL 4는 DevOps, Agile과의 공존을 명시적으로 설계했습니다. |
| ISO 20000 인증이 필요한 경우는? | 서비스 제공업체로서 고객에게 ITSM 역량을 증명해야 할 때 유용합니다. |
