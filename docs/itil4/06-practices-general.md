## 1. 개요

**일반 관리 프랙티스 (General Management Practices)**는 IT 서비스 관리에 한정되지 않고, 비즈니스 전반에서 적용 가능한 경영 관리 프랙티스이다. 총 **14개**의 프랙티스로 구성된다.

| 항목 | 설명 |
|------|------|
| 적용 범위 | IT뿐 아니라 비즈니스 전반 |
| 개수 | 14개 |
| 기원 | 일반 경영 관리 분야에서 채택 |
| ITIL v3 대비 | 일부 신규 추가 (아키텍처 관리 등) |

---

## 2. 전략 관리 (Strategy Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 목표를 정의하고, 목표 달성을 위한 방향과 자원 배분을 결정 |
| 핵심 활동 | 환경 분석, 전략 수립, 전략 실행, 전략 평가 |
| 도구/기법 | SWOT 분석, BCG 매트릭스, 시장 분석 |
| 연관 가치 사슬 | 계획 (Plan) |

---

## 3. 포트폴리오 관리 (Portfolio Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 서비스/제품/프로젝트 포트폴리오가 전략적 목표에 정렬되도록 관리 |
| 핵심 활동 | 포트폴리오 분석, 투자 결정, 우선순위 설정, 성과 추적 |
| 포트폴리오 유형 | 서비스 포트폴리오, 프로젝트 포트폴리오, 고객 포트폴리오 |
| 연관 가치 사슬 | 계획 (Plan) |

---

## 4. 아키텍처 관리 (Architecture Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 아키텍처(비즈니스, 정보, 기술, 서비스)에 대한 이해를 제공하고 효과적 변화 지원 |
| 핵심 활동 | 현재 아키텍처 문서화, 목표 아키텍처 설계, 갭 분석, 로드맵 수립 |
| 아키텍처 유형 | 비즈니스, 서비스, 정보시스템, 기술, 환경 아키텍처 |
| 프레임워크 | TOGAF, Zachman 등 활용 가능 |

---

## 5. 서비스 재무 관리 (Service Financial Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 IT 서비스에 대한 효과적인 예산 편성, 회계, 과금을 지원 |
| 핵심 활동 | 예산 편성(Budgeting), 회계(Accounting), 과금(Charging) |
| 핵심 개념 | TCO(총소유비용), ROI(투자수익률), 비용 모델 |
| 연관 가치 사슬 | 계획 (Plan) |

---

## 6. 인력 및 역량 관리 (Workforce and Talent Management)

| 항목 | 설명 |
|------|------|
| 목적 | 올바른 역량을 갖춘 인력이 올바른 역할에서 효과적으로 일하도록 보장 |
| 핵심 활동 | 인력 계획, 채용, 교육/개발, 성과 관리, 유지 |
| 역량 프레임워크 | SFIA (Skills Framework for the Information Age) |
| 핵심 고려사항 | 기술 격차 분석, 경력 개발 경로, 동기부여 |

---

## 7. 지속적 개선 (Continual Improvement)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 프랙티스와 서비스를 현재 비즈니스 요구에 맞게 지속적으로 정렬하고 개선 |
| 핵심 활동 | 개선 기회 식별, 우선순위화, 개선 실행, 결과 측정 |
| 핵심 도구 | 지속적 개선 레지스터 (CIR), 지속적 개선 모델 |
| 방법론 | PDCA (Plan-Do-Check-Act), Lean, Six Sigma |

### 지속적 개선 모델 7단계

| 단계 | 질문 | 활동 |
|------|------|------|
| 1 | 비전은 무엇인가? | 비즈니스 비전과 목표 확인 |
| 2 | 현재 어디에 있는가? | 현재 상태 기준선 수립 |
| 3 | 어디에 도달하고 싶은가? | 측정 가능한 목표 설정 |
| 4 | 어떻게 도달할 것인가? | 개선 계획 수립 |
| 5 | 조치를 취하라 | 개선 실행 |
| 6 | 도달했는가? | 결과 측정, 목표 대비 평가 |
| 7 | 추진력을 유지하라 | 성과 공유, 문화 정착 |

---

## 8. 측정 및 보고 (Measurement and Reporting)

| 항목 | 설명 |
|------|------|
| 목적 | 의사결정을 지원하고 지속적 개선을 이끌기 위한 정보를 제공 |
| 핵심 활동 | KPI 정의, 데이터 수집, 분석, 보고, 대시보드 운영 |
| 핵심 개념 | CSF (핵심 성공 요인), KPI (핵심 성과 지표), 메트릭 (Metric) |
| 원칙 | 측정은 행동을 유도한다 - 측정 대상을 신중히 선택 |

---

## 9. 리스크 관리 (Risk Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직이 리스크를 효과적으로 이해하고 처리하도록 보장 |
| 핵심 활동 | 리스크 식별, 평가, 대응 전략 수립, 모니터링 |
| 리스크 대응 | 회피(Avoid), 완화(Mitigate), 전가(Transfer), 수용(Accept) |
| 프레임워크 | ISO 31000, COSO ERM |

---

## 10. 정보보안 관리 (Information Security Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 정보에 대한 기밀성, 무결성, 가용성 (CIA)을 보호 |
| 핵심 활동 | 보안 정책 수립, 위협 평가, 보안 통제, 보안 인식 교육 |
| CIA 삼각형 | 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability) |
| 표준 | ISO 27001/27002, NIST Cybersecurity Framework |

---

## 11. 지식 관리 (Knowledge Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직 전체에서 지식을 효과적으로 공유하고 활용하도록 보장 |
| 핵심 활동 | 지식 수집, 저장, 공유, 활용, 갱신 |
| DIKW 모델 | 데이터(Data) → 정보(Information) → 지식(Knowledge) → 지혜(Wisdom) |
| 도구 | 지식 관리 시스템 (KMS), Wiki, FAQ |

---

## 12. 조직 변경 관리 (Organizational Change Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 변화를 원활하고 성공적으로 실행하도록 보장 |
| 핵심 활동 | 변화 준비, 이해관계자 참여, 저항 관리, 교육, 강화 |
| 주의 | IT 변경 관리(Change Enablement)와 다른 개념 |
| 모델 | Kotter의 8단계, ADKAR, Lewin의 변화 모델 |

### Change Enablement vs Organizational Change Management

| 구분 | Change Enablement | Organizational Change Management |
|------|-------------------|--------------------------------|
| 대상 | IT 인프라, 서비스, 구성 변경 | 사람, 문화, 조직 구조 변화 |
| 범위 | 기술적 변경의 리스크 관리 | 조직적 변화의 수용과 정착 |
| 예시 | 서버 패치 적용, 애플리케이션 업그레이드 | 새 ITSM 도구 도입에 따른 업무 방식 변화 |

## 13. 프로젝트 관리 (Project Management)

| 항목 | 설명 |
|------|------|
| 목적 | 프로젝트가 성공적으로 납기, 예산, 범위 내에서 완료되도록 보장 |
| 핵심 활동 | 계획, 위임, 모니터링, 통제, 보고 |
| 방법론 | Waterfall (PRINCE2), Agile (Scrum, Kanban), Hybrid |
| 핵심 제약 | 범위(Scope), 일정(Time), 비용(Cost), 품질(Quality) |

---

## 14. 관계 관리 (Relationship Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직과 이해관계자 간의 관계를 전략적, 전술적 수준에서 수립하고 육성 |
| 핵심 활동 | 이해관계자 식별, 관계 수립, 만족도 관리, 커뮤니케이션 |
| 대상 | 고객, 사용자, 공급자, 내부 팀, 경영진 |
| 핵심 지표 | 고객 만족도 (CSAT), 순추천지수 (NPS) |

---

## 15. 공급자 관리 (Supplier Management)

| 항목 | 설명 |
|------|------|
| 목적 | 조직의 공급자와 그들의 성과가 적절히 관리되어 제품/서비스 품질을 보장 |
| 핵심 활동 | 공급자 평가/선정, 계약 관리, 성과 관리, 관계 관리 |
| 핵심 산출물 | 공급자 데이터베이스, 계약서, 성과 보고서 |
| 고려사항 | 벤더 종속(Vendor Lock-in) 방지, 멀티 소싱 전략 |
