## 1. ITIL이란?

**ITIL (Information Technology Infrastructure Library)**은 IT 서비스 관리(ITSM)를 위한 세계적으로 가장 널리 채택된 프레임워크이다. 영국 정부 기관인 CCTA(Central Computer and Telecommunications Agency)에서 처음 개발되었으며, 현재는 AXELOS(PeopleCert 소유)가 관리한다.

ITIL은 조직이 IT 서비스를 통해 비즈니스 가치를 창출하고 전달하는 데 필요한 **모범 사례(Best Practices)**를 체계화한 가이드라인이다.

| 항목 | 설명 |
|------|------|
| 정식 명칭 | Information Technology Infrastructure Library |
| 최신 버전 | ITIL 5 (2026년 출시), ITIL 4 (2019년 출시) |
| 관리 기관 | PeopleCert (구 AXELOS) |
| 적용 분야 | IT 서비스 관리 (IT Service Management) |
| 특징 | 프레임워크 (Framework), 표준이 아닌 가이드라인 |

## 2. ITIL의 역사

ITIL은 약 30년에 걸쳐 지속적으로 발전해왔다.

| 버전 | 출시 연도 | 주요 특징 |
|------|----------|----------|
| ITIL v1 | 1989 | 최초 출시, 40여 권의 책으로 구성, IT 운영 중심 |
| ITIL v2 | 2001 | 9권으로 통합, 서비스 지원(Service Support)과 서비스 제공(Service Delivery) 중심 |
| ITIL v3 | 2007 | 5권으로 재구성, 서비스 생명주기(Service Lifecycle) 도입 |
| ITIL 2011 Edition | 2011 | v3의 개정판, 프로세스 명확화 및 오류 수정 |
| ITIL 4 | 2019 | 서비스 가치 시스템(SVS) 도입, Agile/DevOps/Lean 통합 |
| ITIL 5 | 2026 | 제품 및 서비스 라이프사이클(PSLM) 도입, AI 거버넌스, 경험 관리 |

### ITIL v3의 5단계 서비스 생명주기

| 단계 | 영문명 | 설명 |
|------|--------|------|
| 서비스 전략 | Service Strategy | 시장 분석, 서비스 포트폴리오 관리 |
| 서비스 설계 | Service Design | 서비스 및 프로세스 설계 |
| 서비스 전환 | Service Transition | 변경 관리, 배포, 테스트 |
| 서비스 운영 | Service Operation | 일상적 서비스 운영 및 지원 |
| 지속적 서비스 개선 | Continual Service Improvement | 서비스 품질 지속적 향상 |

## 3. ITIL v3 vs ITIL 4 주요 차이

| 비교 항목 | ITIL v3 / 2011 | ITIL 4 |
|----------|----------------|--------|
| 핵심 구조 | 5단계 서비스 생명주기 (Service Lifecycle) | 서비스 가치 시스템 (SVS) |
| 구성 요소 | 26개 프로세스 + 4개 기능 | 34개 프랙티스 (Practices) |
| 접근 방식 | 프로세스 중심 (Process-driven) | 가치 중심 (Value-driven) |
| 외부 방법론 | 제한적 통합 | Agile, DevOps, Lean 적극 통합 |
| 흐름 | 선형적 (Linear) | 비선형적, 유연한 가치 사슬 |
| 용어 변화 | 프로세스 (Process) | 프랙티스 (Practice) |
| 변경 관리 | Change Management | Change Enablement (변경 지원) |
| 거버넌스 | 암묵적 | 명시적 거버넌스 구성요소 |
| 자동화 | 부분적 언급 | 지도 원칙으로 강조 |
| 고객 경험 | 서비스 중심 | 가치 공동 창출 (Value Co-creation) |

## 4. ITIL 4 핵심 용어

| 용어 (한국어) | 영문 | 정의 |
|-------------|------|------|
| 서비스 | Service | 고객이 특정 비용과 리스크를 관리하지 않고도 원하는 결과를 달성할 수 있도록 가치를 공동 창출하는 수단 |
| 가치 | Value | 서비스의 인지된 이점, 유용성, 중요성 |
| 결과 | Outcome | 이해관계자가 달성하고자 하는 결과 또는 변화 |
| 산출물 | Output | 활동의 유형적(tangible) 또는 무형적(intangible) 결과물 |
| 비용 | Cost | 서비스를 제공하거나 소비하는 데 들어가는 금전적 비용 |
| 리스크 | Risk | 불확실성에 의해 발생할 수 있는 가능한 사건 |
| 유틸리티 | Utility | 서비스가 제공하는 기능성, "무엇을 하는가" (Fit for Purpose) |
| 보증 | Warranty | 서비스의 사용 적합성, "어떻게 수행되는가" (Fit for Use) |
| 서비스 관계 | Service Relationship | 서비스 제공자와 소비자 간의 협력 관계 |
| 가치 공동 창출 | Value Co-creation | 서비스 제공자와 소비자가 함께 가치를 만들어내는 과정 |
| 서비스 제공 | Service Provision | 서비스 제공자가 수행하는 서비스 관리 활동 |
| 서비스 소비 | Service Consumption | 서비스 소비자가 수행하는 서비스 관리 활동 |

### 유틸리티(Utility)와 보증(Warranty)의 관계

| 구분 | 유틸리티 (Utility) | 보증 (Warranty) |
|------|-------------------|----------------|
| 질문 | "서비스가 무엇을 하는가?" | "서비스가 충분히 잘 수행되는가?" |
| 의미 | 목적 적합성 (Fit for Purpose) | 사용 적합성 (Fit for Use) |
| 관심사 | 기능, 제약 제거 | 가용성, 용량, 연속성, 보안 |
| 조건 | 둘 다 충족해야 가치 생성 | 둘 다 충족해야 가치 생성 |

## 5. ITIL 4의 목표

ITIL 4는 다음과 같은 목표를 추구한다.

| 목표 | 설명 |
|------|------|
| 가치 공동 창출 | 서비스 제공자와 소비자가 함께 비즈니스 가치를 창출 |
| 현대적 방법론 통합 | Agile, DevOps, Lean, Cloud 등 현대적 IT 접근법과 조화 |
| 유연한 프레임워크 | 조직의 규모와 상황에 맞게 적응 가능한 유연한 구조 제공 |
| 전체적 관점 | 기술뿐 아니라 사람, 파트너, 프로세스를 포함한 총체적 접근 |
| 지속적 개선 문화 | 끊임없는 개선을 조직 문화로 정착시키는 것을 지향 |
| 디지털 전환 지원 | 디지털 전환 시대에 맞는 IT 서비스 관리 체계 제공 |
