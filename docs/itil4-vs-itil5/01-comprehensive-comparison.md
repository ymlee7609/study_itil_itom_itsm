## 1. 한눈에 보는 비교

| 비교 항목 | ITIL 4 (2019) | ITIL 5 (2026) |
|----------|---------------|---------------|
| **핵심 초점** | IT 서비스 관리 (ITSM) | 디지털 제품 및 서비스 관리 (DPSM) |
| **라이프사이클** | 서비스 가치 사슬 (SVC) 6개 활동 | 제품 및 서비스 라이프사이클 (PSLM) 8개 활동 |
| **가치 시스템** | 서비스 가치 시스템 (SVS) | ITIL 가치 시스템 (ITIL Value System) |
| **가이딩 원칙** | 7개 (동일) | 7개 (동일) |
| **4가지 차원** | 동일 | 동일 (AI 보강) |
| **프랙티스** | 34개 | 34개 (용어 현대화) |
| **AI** | 자동화의 일부로 간접 언급 | AI 거버넌스 공식 프레임워크 + 6C 모델 |
| **거버넌스** | 작업 위에서 통제 | 작업 곁에서 가이드 |
| **경험 관리** | 가치 공동 창출 개념 | XLA, 경험 중심 성과 평가 |
| **서비스 관계** | 기본 개념 | 서비스 관계 모델 (Basic/Cooperative/Collaborative) |
| **콘텐츠** | - | 36% 신규 + 64% 발전 |
| **인증 모듈** | 5개 핵심 | 9개 핵심 + 1개 확장 (AI Governance) |

## 2. 프레임워크 구조 비교

### ITIL 4 구조

```
┌─────────────────────────────────────────────┐
│           서비스 가치 시스템 (SVS)             │
├─────────────────────────────────────────────┤
│  기회/수요 →                                 │
│  ┌─────────────────────────────────────┐    │
│  │    7대 가이딩 원칙                    │    │
│  ├─────────────────────────────────────┤    │
│  │    거버넌스                           │    │
│  ├─────────────────────────────────────┤    │
│  │    서비스 가치 사슬 (SVC)              │    │
│  │    Plan → Engage → Design/Transition │    │
│  │    → Obtain/Build → Deliver/Support  │    │
│  │    → Improve                         │    │
│  ├─────────────────────────────────────┤    │
│  │    34개 프랙티스                      │    │
│  ├─────────────────────────────────────┤    │
│  │    지속적 개선                        │    │
│  └─────────────────────────────────────┘    │
│                                → 가치       │
└─────────────────────────────────────────────┘
```

### ITIL 5 구조

```
┌─────────────────────────────────────────────────┐
│           ITIL 가치 시스템 (ITIL Value System)    │
├─────────────────────────────────────────────────┤
│  기회/수요 →                                     │
│  ┌─────────────────────────────────────────┐    │
│  │    7대 가이딩 원칙 (동일)                  │    │
│  ├─────────────────────────────────────────┤    │
│  │    임베디드 거버넌스 (변화)                 │    │
│  ├─────────────────────────────────────────┤    │
│  │    ITIL 가치 사슬 / PSLM (확장)           │    │
│  │    Discover → Design → Acquire → Build   │    │
│  │    → Transition → Operate → Deliver      │    │
│  │    → Support                             │    │
│  ├─────────────────────────────────────────┤    │
│  │    34개 프랙티스 (용어 현대화)             │    │
│  ├─────────────────────────────────────────┤    │
│  │    AI 거버넌스 + 6C 모델 (신규)           │    │
│  ├─────────────────────────────────────────┤    │
│  │    지속적 개선 (전체 내재화)               │    │
│  └─────────────────────────────────────────┘    │
│                                   → 가치/경험    │
└─────────────────────────────────────────────────┘
```

## 3. 핵심 모델 비교

### 서비스 가치 사슬 → 제품 및 서비스 라이프사이클

| ITIL 4 (6개 활동) | ITIL 5 (8개 활동) | 변화 설명 |
|-------------------|-------------------|-----------|
| Plan | **Discover** | 계획 → 발견으로 확장. 니즈/기회 식별 포함 |
| Engage | *(관계 모델로 분리)* | 별도 서비스 관계 모델로 체계화 |
| Design & Transition | **Design** + **Transition** | 하나의 활동이 두 개로 분리 |
| Obtain/Build | **Acquire** + **Build** | 획득과 구축이 독립 활동으로 분리 |
| Deliver & Support | **Deliver** + **Support** | 전달과 지원이 독립 활동으로 분리 |
| Improve | *(전체 활동에 내재화)* | 별도 활동 → 모든 활동에 포함 |
| - | **Operate** | 일상 운영이 독립 활동으로 신설 |

### 핵심 차이점

- ITIL 4는 합쳐진 활동(예: Design & Transition)을 하나로 다뤘지만, ITIL 5는 각각을 독립 활동으로 분리하여 더 세밀한 관리 가능
- Improve가 독립 활동에서 전체 활동에 내재화됨 → 개선이 특정 단계가 아닌 **상시 활동**
- Operate가 신설되어 일상 운영이 명시적으로 강조됨

## 4. 거버넌스 비교

| 비교 항목 | ITIL 4 | ITIL 5 |
|----------|--------|--------|
| 위치 | 작업 위(Above) | 작업 곁(Beside) |
| 역할 | 통제/승인 게이트 | 가드레일/가이드라인 |
| CAB (변경 자문 위원회) | 모든 변경에 대해 검토 | 기준 내 변경은 자동 승인 |
| 정책 결정 | 중앙 거버넌스 팀 | 원칙 기반 분산 자율 |
| 리뷰 주기 | 정기적 (분기/월) | 지속적 모니터링 |
| DevOps 호환성 | 부분적 충돌 가능 | 자연스러운 통합 |

## 5. AI와 기술 비교

| 비교 항목 | ITIL 4 | ITIL 5 |
|----------|--------|--------|
| AI 다룸 | "최적화하고 자동화하라" 원칙 내 간접 언급 | 독립 프레임워크: AI 거버넌스 |
| AI 역량 분류 | 없음 | 6C 모델 (Creation/Curation/Clarification/Cognition/Communication/Coordination) |
| AI 윤리 | 언급 없음 | 윤리적 AI 도입 가이드라인 |
| AI 인증 | 없음 | ITIL AI Governance (확장 모듈) |
| 자동화 범위 | IT 프로세스 자동화 | AI 기반 지능형 자동화 포함 |
| 클라우드 | 부분적 언급 | 클라우드 네이티브 기본 전제 |
| 데이터 거버넌스 | 정보 보안 관리 내 포함 | AI 데이터 거버넌스까지 확장 |

## 6. 인증 체계 비교

### 구조 비교

| 레벨 | ITIL 4 | ITIL 5 |
|------|--------|--------|
| 입문 | Foundation | Foundation (AI 기초 추가) |
| 실무 | CDS, DSV, HVIT | MSF, PIC, CAI |
| 전문 지식 | - | Product, Service, Experience, Strategy |
| 공통 필수 | DPI | Transformation |
| 전략 | DITS | Strategy |
| AI | 없음 | AI Governance (확장) |
| PM | 없음 | Practice Manager (신규 지정) |
| MP | CDS+DSV+HVIT+DPI | Product+Service+Experience+Transformation |
| SL | DPI+DITS | Strategy+Transformation |
| Master | MP+SL | PM+MP+SL |

### 인증 전환

| ITIL 4 보유 | ITIL 5 전환 |
|------------|------------|
| Foundation | 그대로 인정, 재시험 불요 |
| MP 일부 | 모듈별 크로스 인정 확인 필요 |
| MP/SL 완료 | 전환 경로 추후 발표 |

## 7. 실무 영향 비교

### 서비스 데스크 관점

| 항목 | ITIL 4 | ITIL 5 |
|------|--------|--------|
| 성과 지표 | MTTR, SLA 준수율 | + CSAT, NPS, CES, XLA |
| AI 활용 | 선택적 자동화 | AI 챗봇, 자동 분류, 예측적 지원 |
| 사용자 경험 | 부가적 고려 | 핵심 성과 지표 |

### IT 운영 관점

| 항목 | ITIL 4 | ITIL 5 |
|------|--------|--------|
| 모니터링 | 이벤트 관리 | + AI 기반 이상 탐지, 예측적 운영 |
| 변경 관리 | CAB 중심 승인 | 기준 내 자동 승인, 임베디드 거버넌스 |
| 서비스 관점 | 서비스 중심 | 제품+서비스 통합 관점 |

### IT 전략 관점

| 항목 | ITIL 4 | ITIL 5 |
|------|--------|--------|
| 전략 범위 | IT 전략 ↔ 비즈니스 전략 정렬 | + AI 전략, 디지털 제품 전략 포함 |
| 거버넌스 | 중앙 집중식 | 분산 자율 + 중앙 원칙 |
| 혁신 | 디지털 전환 지원 | AI/클라우드 기반 지속적 혁신 |

---

## 8. 전환 전략

### 조직 관점

| 전환 단계 | 활동 |
|----------|------|
| **1. 현황 파악** | 현재 ITIL 성숙도 평가, ITIL 5 갭 분석 |
| **2. 영향 분석** | PSLM 도입 영향, AI 거버넌스 필요성 평가, 경험 관리 도입 범위 결정 |
| **3. 교육 계획** | 대상별 인증 경로 수립, 교육 예산 확보 |
| **4. 점진적 적용** | 파일럿 프로젝트 → 확산, 급격한 전환보다 점진적 채택 |
| **5. 지속적 개선** | ITIL 5의 내재화된 개선 원칙에 따라 지속적 최적화 |

### 개인 관점

| 현재 상태 | 추천 행동 |
|----------|----------|
| ITIL 미경험 | ITIL 5 Foundation 취득 → 관심 트랙 선택 |
| ITIL 4 Foundation | ITIL 5 차이점 학습 → 고급 모듈 선택적 취득 |
| ITIL 4 MP/SL | 전환 경로 발표 대기, ITIL 5 신규 개념(AI, 경험, 제품) 학습 |
| IT 운영 실무자 | Foundation → PM 경로 (MSF + Transformation) |
| IT 관리자/리더 | Foundation → SL 경로 (Strategy + Transformation) |
| AI/디지털 전문가 | Foundation + AI Governance |
