# ITAM vs ITSM vs ITOM 비교

## 1. 한눈에 보는 비교

### 레스토랑 비유로 이해하기

```
┌──────────────────────────────────────────────────────────────┐
│                    🏢 IT 조직 = 레스토랑                       │
├──────────────┬──────────────┬──────────────┬────────────────┤
│    ITIL 4    │    ITSM      │    ITOM      │     ITAM       │
│              │              │              │                │
│  "요리책"     │ "홀 서비스"   │ "주방 운영"   │ "식재료/장비   │
│              │              │              │  재고 관리"     │
│              │              │              │                │
│ 최고의 서비스 │ 손님 주문을   │ 주방 설비가    │ 어떤 장비가    │
│ 를 위한      │ 받고, 요리를  │ 잘 돌아가는지  │ 몇 개 있고,   │
│ 모범 사례    │ 전달하고,     │ 모니터링하고   │ 상태는 어떤지  │
│ 모음집       │ 피드백 수집   │ 문제를 예방    │ 추적/관리      │
│              │              │              │                │
│ "어떻게 해야 │ "손님 만족"   │ "주방이 멈추지│ "재고 파악과   │
│  하는가?"    │  이 최우선    │  않게 한다"   │  비용 절감"    │
└──────────────┴──────────────┴──────────────┴────────────────┘
```

## 2. 상세 비교표

| 구분 | ITIL 4 | ITSM | ITOM | ITAM |
|------|--------|------|------|------|
| **정의** | IT 서비스 관리 모범 사례 프레임워크 | IT 서비스의 설계, 제공, 관리 | IT 인프라의 일상 운영 관리 | IT 자산의 전체 생애주기 관리 |
| **초점** | "어떻게" 해야 하는가 | 서비스 제공 (사용자 대면) | 인프라 운영 (백엔드) | 자산의 재무/계약/물리적 관리 |
| **핵심 질문** | "모범 사례는 무엇인가?" | "사용자에게 서비스를 잘 제공하고 있나?" | "시스템이 안정적으로 운영되고 있나?" | "어떤 자산이 있고 비용은 적절한가?" |
| **주요 활동** | 프랙티스 정의, 가이드라인 | 인시던트/문제/변경 관리 | 모니터링, 이벤트, 디스커버리 | 자산 추적, 라이선스 관리, 폐기 |
| **수혜자** | IT 조직 전체 | 서비스 사용자 (직원/고객) | IT 운영팀 | 재무팀, IT 관리자, 감사팀 |
| **KPI 예시** | 프랙티스 성숙도 | SLA 준수율, MTTR | 가용성, 장애 건수 | 자산 활용률, 라이선스 준수율 |

## 3. 관계도: 어떻게 연결되는가?

```
┌─────────────────────────────────────────────────────────────┐
│                      ITIL 4 Framework                        │
│               (모든 IT 관리의 모범 사례 가이드)                 │
│                                                              │
│  ┌─────────────┐  ┌──────────────┐  ┌─────────────────┐     │
│  │    ITSM     │  │    ITOM      │  │      ITAM       │     │
│  │             │  │              │  │                 │     │
│  │ 서비스 요청 ◄──► 인프라 상태  ◄──► 자산 정보       │     │
│  │ 인시던트    │  │ 이벤트       │  │ 라이선스        │     │
│  │ 변경 관리   │  │ 모니터링     │  │ 재고/비용       │     │
│  └──────┬──────┘  └──────┬───────┘  └────────┬────────┘     │
│         │                │                    │              │
│         └────────────────┼────────────────────┘              │
│                          │                                   │
│                    ┌─────▼─────┐                             │
│                    │   CMDB    │                             │
│                    │ (통합 DB) │                             │
│                    └───────────┘                             │
└─────────────────────────────────────────────────────────────┘
```

### 연결 포인트

| 연결 | 설명 | 실무 예시 |
|------|------|-----------|
| **ITSM ↔ ITAM** | 서비스 요청 시 자산 정보 활용 | "새 노트북 요청" → ITAM에서 재고 확인 후 배치 |
| **ITOM ↔ ITAM** | 운영 모니터링에서 자산 상태 확인 | 서버 장애 감지 → ITAM에서 보증 상태/교체 시기 확인 |
| **ITSM ↔ ITOM** | 인시던트 발생 시 운영 데이터 참조 | 사용자 장애 신고 → ITOM 모니터링 데이터로 원인 분석 |
| **CMDB** | 세 영역의 데이터가 통합되는 중심 | 자산, 서비스, 인프라 정보의 단일 진실의 원천 |

## 4. 실무 시나리오로 이해하기

### 시나리오 1: 직원 입사

```
[신규 직원 입사]
     │
     ▼
ITSM: 서비스 요청 접수 (노트북, 소프트웨어 필요)
     │
     ▼
ITAM: 재고 확인 → 노트북 5대, MS365 라이선스 8개 남음
     │            → 새로 구매하지 않고 기존 재고에서 배치!
     ▼
ITOM: 배치된 장비를 모니터링 대상에 추가
     │
     ▼
ITSM: 서비스 요청 완료 처리, 사용자에게 안내
```

### 시나리오 2: 서버 장애

```
[서버 응답 지연 감지]
     │
     ▼
ITOM: 모니터링 알림 발생 → 디스크 용량 90% 초과 확인
     │
     ▼
ITSM: 인시던트 자동 생성 → 담당자 배정
     │
     ▼
ITAM: 해당 서버 자산 정보 조회
     │    → 보증 기간: 2개월 남음
     │    → 사용 연한: 4년 (교체 주기 도래)
     ▼
의사결정: 임시 조치 후 자산 교체 계획 수립
```

### 시나리오 3: 소프트웨어 감사

```
[라이선스 감사 통보 수신]
     │
     ▼
ITAM: 보유 라이선스 vs 실제 설치 현황 비교
     │    → 100개 라이선스 보유, 87개 사용 중
     │    → 13개 미사용 라이선스 확인
     ▼
ITOM: 실제 소프트웨어 설치 현황 디스커버리로 검증
     │
     ▼
ITSM: 감사 대응 보고서 생성, 미사용 라이선스 회수 프로세스 시작
```

## 5. 조직에 설명할 때 핵심 메시지

### 경영진에게

> "ITIL은 **규칙서**, ITSM은 **고객 서비스**, ITOM은 **공장 관리**, ITAM은 **재고/비용 관리**입니다. 네 가지가 함께 돌아가야 IT가 비용 효율적이고 안정적으로 운영됩니다."

### IT 부서에게

> "ITSM으로 사용자 요청을 처리하고, ITOM으로 인프라를 안정적으로 운영하고, ITAM으로 자산과 비용을 추적합니다. ITIL 4가 이 모든 활동의 모범 사례를 제공합니다."

### 현업 부서에게

> "여러분이 IT에 뭔가 요청하면 그게 **ITSM**, IT가 시스템이 안 멈추도록 관리하는 게 **ITOM**, 여러분에게 노트북과 소프트웨어를 지급하고 추적하는 게 **ITAM**입니다."
