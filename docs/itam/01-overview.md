# ITAM (IT Asset Management) 개요

## 1. ITAM이란?

IT 자산 관리(ITAM, IT Asset Management)는 조직이 보유한 모든 IT 자산의 **구매, 배치, 사용, 유지보수, 폐기**까지 전체 생애주기를 체계적으로 추적하고 관리하는 프로세스입니다.

### 핵심 정의

> ITAM은 IT 자산의 **재정적, 계약적, 물리적** 측면을 통합 관리하여 비용을 최적화하고 리스크를 줄이는 활동입니다.

### IT 자산의 범위

| 자산 유형 | 예시 |
|-----------|------|
| **하드웨어 자산** | 노트북, 데스크톱, 서버, 네트워크 장비, 모바일 기기 |
| **소프트웨어 자산** | 라이선스, SaaS 구독, 온프레미스 소프트웨어 |
| **클라우드 자산** | AWS/Azure/GCP 인스턴스, 스토리지, 서비스 |
| **디지털 자산** | SSL 인증서, 도메인, IP 주소 |

---

## 2. ITAM이 필요한 이유

### 비용 문제

- 기업의 약 **50%가 소프트웨어/SaaS 지출의 10% 이상**을 낭비
- 미사용 소프트웨어 라이선스로 인한 연간 비용 낭비가 수억 원에 달함
- 클라우드 리소스의 약 **53%가 비관리/미사용 상태**로 과잉 지출 발생

### 리스크 문제

- 라이선스 미준수 시 법적 제재 및 벌금
- 관리되지 않는 자산은 보안 취약점의 원인
- 자산 현황 미파악으로 인한 비효율적 의사결정

---

## 3. ITAM의 3대 핵심 영역

### 3.1 하드웨어 자산 관리 (HAM)

- 물리적 IT 장비의 수명주기 관리
- 재고 추적 및 배치 관리
- 유지보수 일정 및 교체 계획

### 3.2 소프트웨어 자산 관리 (SAM)

- 라이선스 준수(Compliance) 관리
- 소프트웨어 사용량 모니터링
- SaaS 구독 최적화

### 3.3 클라우드 자산 관리

- 클라우드 인스턴스 및 서비스 추적
- 비용 최적화 (FinOps)
- 멀티클라우드 환경 통합 관리

---

## 4. ITAM 생애주기

```
요청 → 조달 → 배치 → 운영/유지보수 → 회수 → 폐기/재활용
 │       │       │         │            │        │
 ▼       ▼       ▼         ▼            ▼        ▼
예산   구매/    설치/     패치/       반납/    데이터
승인   계약    구성     업그레이드   재배치    삭제
```

### 각 단계별 핵심 활동

| 단계 | 핵심 활동 | 담당 |
|------|-----------|------|
| **요청** | 자산 필요성 평가, 예산 승인 | 사용자 + 관리자 |
| **조달** | 구매, 계약, 라이선스 확보 | 구매팀 + IT |
| **배치** | 설치, 구성, 사용자 할당 | IT 운영팀 |
| **운영** | 모니터링, 유지보수, 업데이트 | IT 운영팀 |
| **회수** | 반납, 재배치, 재활용 검토 | IT + 자산관리팀 |
| **폐기** | 데이터 삭제, 환경 규정 준수 처리 | IT + 법무 |

---

## 5. ITAM의 비즈니스 가치

### 정량적 효과

- **비용 절감**: 연간 IT 예산의 25~30% 절감 효과
- **라이선스 최적화**: 미사용 라이선스 회수로 불필요한 구매 방지
- **감사 대응**: 소프트웨어 라이선스 감사 시 즉각 대응 가능

### 정성적 효과

- **보안 강화**: 관리되지 않는 "그림자 IT" 자산 가시성 확보
- **의사결정 지원**: 정확한 자산 데이터 기반의 투자 결정
- **규정 준수**: 라이선스 계약 및 규정 자동 추적
- **ESG 대응**: IT 자산의 환경 영향 추적 및 관리

---

## 6. ITAM과 ITIL 4의 관계

ITIL 4에서 ITAM은 **"IT 자산 관리(IT Asset Management)"** 프랙티스로 정의되어 있으며, 서비스 관리 프랙티스 중 하나입니다.

### ITIL 4 프랙티스로서의 ITAM

- **목적**: IT 자산의 전체 생애주기를 계획하고 관리
- **범위**: 하드웨어, 소프트웨어, 클라우드, 데이터를 포함
- **연관 프랙티스**: 서비스 구성 관리(CMDB), 재무 관리, 변경 관리

## 7. 2025-2026년 ITAM 트렌드

| 트렌드 | 설명 |
|--------|------|
| **AI/ML 기반 자산 관리** | AI를 활용한 자산 수명 예측, 자동 최적화 |
| **FinOps 통합** | 클라우드 비용 관리와 ITAM의 융합 |
| **Zero Trust 통합** | 자산 기반 보안 정책 자동 적용 |
| **ESG 연계** | IT 자산의 탄소 발자국 추적 |
| **SaaS 관리 강화** | 급증하는 SaaS 구독의 통합 가시성 |
