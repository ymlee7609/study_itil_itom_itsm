## 1. 인증 체계 개요

ITIL 5 인증 체계는 **9개 핵심 모듈 + 1개 확장 모듈**로 구성된다.

```
                        ┌──────────────┐
                        │    Master    │
                        └──────┬───────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
     ┌────────┴────────┐ ┌────┴─────┐ ┌────────┴────────┐
     │   Practice      │ │ Managing │ │   Strategic     │
     │   Manager (PM)  │ │ Prof (MP)│ │   Leader (SL)   │
     └────────┬────────┘ └────┬─────┘ └────────┬────────┘
              │               │                │
     ┌────────┴────────┐     │       ┌────────┴────────┐
     │ MSF / PIC / CAI │     │       │    Strategy     │
     │   (1개 선택)     │     │       └────────┬────────┘
     └────────┬────────┘     │                │
              │        ┌─────┴──────┐         │
              │        │  Product   │         │
              │        │  Service   │         │
              │        │ Experience │         │
              │        └─────┬──────┘         │
              │              │                │
              └──────────────┼────────────────┘
                    ┌────────┴────────┐
                    │ Transformation  │ ← 모든 경로의 필수 모듈
                    └────────┬────────┘
                    ┌────────┴────────┐
                    │   Foundation    │
                    └─────────────────┘

     ┌─────────────────────────────────┐
     │  AI Governance (확장 모듈)       │ ← 선수과목 없음, 독립 수강 가능
     └─────────────────────────────────┘
```

## 2. Foundation

| 항목 | 내용 |
|------|------|
| **대상** | ITIL 입문자, 모든 IT 직군 |
| **전제 조건** | 없음 |
| **학습 내용** | ITIL 5 핵심 개념, 7대 가이딩 원칙, SVS, PSLM 8개 활동, 4가지 차원, 34개 프랙티스, AI 거버넌스 기초 |
| **시험 형식** | 객관식 40문항, 60분 |
| **합격 기준** | 65% (26/40 정답) |
| **출시일** | 2026년 2월 12일 |
| **특이사항** | 15개 AI 관련 신규 용어 포함 |

### Foundation에서 다루는 주요 영역

| 영역 | 비중 | 핵심 학습 내용 |
|------|------|-------------|
| 핵심 개념 | 높음 | 서비스, 가치, 유틸리티/보증, 결과/산출물 |
| 가이딩 원칙 | 높음 | 7대 원칙 명칭, 정의, 적용 방법 |
| SVS / PSLM | 높음 | SVS 구성요소, PSLM 8개 활동 |
| 4가지 차원 | 중간 | 4가지 차원 명칭과 핵심 내용 |
| 34개 프랙티스 | 중간 | 프랙티스 분류와 주요 프랙티스 목적 |
| AI 거버넌스 | 낮음 | 6C 모델 인식 수준, 기본 용어 |

## 3. 중급 모듈 (Intermediate)

### 실무 운영 모듈 (Practice-focused)

| 모듈 | 정식 명칭 | 핵심 내용 |
|------|----------|-----------|
| **MSF** | Monitor, Support and Fulfil | 모니터링, 지원, 요청 이행, 서비스 데스크 운영 |
| **PIC** | Plan, Implement and Control | 계획, 구현, 통제, 변경 관리, 릴리즈 관리 |
| **CAI** | Collaborate, Assure and Improve | 협업, 품질 보증, 지속적 개선 |

### 전문 지식 모듈 (Knowledge-focused)

| 모듈 | 정식 명칭 | 핵심 내용 |
|------|----------|-----------|
| **Product** | ITIL Product (Version 5) | 디지털 제품 관리, 제품 라이프사이클, 제품 로드맵 |
| **Service** | ITIL Service (Version 5) | 서비스 설계, 전달, 지원, 서비스 관계 관리 |
| **Experience** | ITIL Experience (Version 5) | 사용자 경험 관리, XLA, 여정 매핑, 인간 중심 설계 |
| **Strategy** | ITIL Strategy (Version 5) | 디지털 전략 수립, IT-비즈니스 정렬, 전략적 의사결정 |

### 필수 공통 모듈

| 모듈 | 정식 명칭 | 핵심 내용 |
|------|----------|-----------|
| **Transformation** | ITIL Transformation (Version 5) | 조직 변화 관리, 디지털 전환, 지속적 개선, 리더십 |

### 확장 모듈

| 모듈 | 정식 명칭 | 핵심 내용 |
|------|----------|-----------|
| **AI Governance** | ITIL AI Governance (Version 5) | AI 거버넌스 프레임워크, 6C 모델 심화, 윤리적 AI |

## 4. 전문 지정 (Professional Designations)

### Practice Manager (PM)

| 항목 | 내용 |
|------|------|
| **대상** | 운영 관리자, 서비스 관리 실무자 |
| **초점** | 운영 탁월성, 프랙티스 적용 |
| **필수 모듈** | MSF/PIC/CAI 중 **1개** + **Transformation** |
| **역량** | ITIL 가치 시스템 전반에 걸친 개선과 변화 설계, 계획, 관리 |

### Managing Professional (MP)

| 항목 | 내용 |
|------|------|
| **대상** | 디지털 제품/서비스의 E2E 관리 담당자 |
| **초점** | 제품/서비스 전체 라이프사이클 관리 |
| **필수 모듈** | **Product** + **Service** + **Experience** + **Transformation** |
| **역량** | 서비스 라이프사이클 전체(생성 → 전달 → 지속적 개선)에 ITIL 원칙 적용 |

### Strategic Leader (SL)

| 항목 | 내용 |
|------|------|
| **대상** | CIO, CTO, IT 전략 리더 |
| **초점** | IT/디지털 전략과 비즈니스 전략 정렬 |
| **필수 모듈** | **Strategy** + **Transformation** |
| **역량** | 전략적 의사결정, 거버넌스, 장기적 가치 창출 |

## 5. ITIL Master

| 항목 | 내용 |
|------|------|
| **전제 조건** | **PM + MP + SL** 모두 취득 |
| **추가 요건** | 실무 경험 (세부 요건 추후 발표 예정) |
| **평가 방식** | 추후 발표 예정 |
| **위상** | ITIL 인증의 최고 레벨 |

## 6. ITIL 4에서 ITIL 5로의 전환

### 핵심 원칙

- ITIL 4 인증은 **계속 유효**하다
- ITIL 4 Foundation은 ITIL 5 고급 모듈의 선수 조건으로 **인정**된다
- Foundation 재시험 없이 바로 ITIL 5 고급 모듈 수강 가능
- 전환 기간 동안 ITIL 4와 ITIL 5 인증이 **병행 운영**된다

### 전환 경로

| 현재 보유 인증 | 전환 방법 |
|-------------|----------|
| ITIL 4 Foundation | 바로 ITIL 5 고급 모듈 수강 가능 (Foundation 재시험 불요) |
| ITIL 4 MP 일부 | ITIL 5 모듈로 나머지 이수 (크로스 인정은 모듈별 확인 필요) |
| ITIL 4 MP 완료 | ITIL 5 MP 전환 경로 추후 발표 예정 |
| ITIL 4 SL 완료 | ITIL 5 SL 전환 경로 추후 발표 예정 |
| 인증 없음 | ITIL 5 Foundation부터 시작 |

### 추천 학습 경로

| 대상 | 추천 경로 |
|------|----------|
| **ITIL 처음 시작** | ITIL 5 Foundation → 관심 트랙 선택 |
| **ITIL 4 Foundation 보유** | ITIL 5 차이점 학습 → 관심 트랙 모듈 수강 |
| **IT 운영/서비스 데스크** | Foundation → MSF → Transformation (PM) |
| **제품/서비스 관리자** | Foundation → Product + Service + Experience + Transformation (MP) |
| **IT 전략 리더** | Foundation → Strategy + Transformation (SL) |
| **AI 관심** | Foundation → AI Governance (독립 수강 가능) |

## 8. 시험 정보

### Foundation 시험

| 항목 | 내용 |
|------|------|
| 문항 수 | 40문항 (객관식) |
| 시험 시간 | 60분 |
| 합격 기준 | 65% (26/40) |
| 오픈 북 | 아니오 (Closed Book) |
| 시험 방식 | 온라인 감독 또는 시험 센터 |
| 시험 언어 | 영어 (추가 언어 순차 지원) |

### 고급 모듈 시험 (공통)

| 항목 | 내용 |
|------|------|
| 전제 조건 | ITIL Foundation (v4 또는 v5) |
| 시험 형식 | 객관식 (시나리오 기반) |
| 합격 기준 | 추후 확정 (ITIL 4 기준 70% 예상) |
| 특이사항 | 시나리오 기반 응용 문제 포함 |

### AI Governance 시험

| 항목 | 내용 |
|------|------|
| 전제 조건 | **없음** (Foundation 없이 수강 가능) |
| 시험 형식 | Closed Book |
| 특이사항 | ITIL 5에서 유일하게 선수과목이 없는 고급 모듈 |

## 9. ITIL 4 vs ITIL 5 인증 비교

| 비교 항목 | ITIL 4 | ITIL 5 |
|----------|--------|--------|
| 총 모듈 수 | 5개 핵심 + Practice 모듈 | 9개 핵심 + 1개 확장 |
| Foundation | 서비스 관리 기초 | 서비스 관리 기초 + AI 거버넌스 기초 |
| MP 경로 | CDS + DSV + HVIT + DPI | Product + Service + Experience + Transformation |
| SL 경로 | DPI + DITS | Strategy + Transformation |
| 공통 모듈 | DPI (MP/SL 공통) | Transformation (모든 경로 공통) |
| PM 경로 | 없음 | MSF/PIC/CAI 중 1개 + Transformation (신규) |
| AI 모듈 | 없음 | AI Governance (확장, 선수조건 없음) |
| Master 조건 | MP + SL | PM + MP + SL |
