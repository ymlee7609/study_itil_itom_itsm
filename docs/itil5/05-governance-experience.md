## 1. 거버넌스 방식 변화

ITIL 5에서의 거버넌스는 근본적으로 **역할이 재정의**되었다. 더 이상 속도를 늦추는 검문소가 아니라, 팀이 자율적으로 운영할 수 있는 **가드레일(Guardrail)**로 작동한다.

### ITIL 4 vs ITIL 5 거버넌스 비교

| 비교 항목 | ITIL 4 | ITIL 5 |
|----------|--------|--------|
| 위치 | 작업 위(Above work) | 작업 곁(Beside work) |
| 역할 | 검토/승인 게이트 | 경계와 가이드라인 제공 |
| 속도 영향 | 종종 속도 저하 | 자율성 증대, 속도 향상 |
| 접근법 | 중앙 집중식 통제 | 분산형 자율 운영 + 중앙 원칙 |
| 비유 | 교차로의 경찰관 | 도로의 가드레일과 표지판 |

### 핵심 변화 포인트

1. **임베디드 거버넌스**: 거버넌스가 별도 프로세스가 아니라, 각 팀의 작업 흐름에 내재된다
2. **원칙 기반 운영**: 세부 규칙이 아닌 원칙(Principles)과 경계(Boundaries)를 설정
3. **피드백 루프 강화**: 거버넌스 결과가 즉시 팀에 피드백되는 구조
4. **DevOps/Agile 정합**: DevOps 팀의 자율적 운영 방식과 자연스럽게 조화

### 실무 영향

| 기존 (ITIL 4 방식) | 변경 (ITIL 5 방식) |
|-------------------|-------------------|
| 모든 변경에 CAB 승인 필요 | 사전 정의된 기준 내 표준 변경은 자동 승인 |
| 중앙 거버넌스 팀이 정책 결정 | 팀이 가이드라인 안에서 자율적 결정 |
| 분기별 거버넌스 리뷰 | 지속적 거버넌스 모니터링 + 자동화 |

## 2. 서비스 관계 모델 (Service Relationship Model)

ITIL 5는 서비스 관계를 세 가지 유형으로 분류하는 **서비스 관계 모델**을 새롭게 도입했다.

### 2.1 서비스 관계의 세 가지 유형

| 유형 | 영문 | 설명 | 예시 |
|------|------|------|------|
| **기본적** | Basic | 표준화된 서비스 제공, 최소한의 상호작용 | SaaS 구독, 표준 인프라 서비스 |
| **협력적** | Cooperative | 상호 이해와 조율을 통한 서비스 관계 | 관리형 서비스 (MSP), 파트너십 기반 아웃소싱 |
| **협업적** | Collaborative | 서비스 제공자와 소비자가 함께 가치를 설계하고 창출 | 공동 개발 프로젝트, 플랫폼 에코시스템 |

### 2.2 이해관계자 역할

ITIL 5는 서비스 소비 측의 역할을 명확히 구분한다.

| 역할 | 영문 | 설명 |
|------|------|------|
| **스폰서** | Sponsor | 서비스 예산과 요구사항을 승인하는 의사결정자 |
| **고객** | Customer | 서비스 요구사항을 정의하고 결과를 평가하는 사람 |
| **사용자** | User | 서비스를 실제로 사용하는 사람 |

### 2.3 서비스 여정 (Service Journey)

서비스 소비자가 서비스와의 관계에서 겪는 전체 경험을 **서비스 여정**으로 정의한다. 이는 고객 여정 매핑(Customer Journey Mapping) 기법과 유사하며, 각 접점(Touchpoint)에서의 경험을 설계하고 측정한다.

```
탐색 → 참여 → 온보딩 → 공동 창출 → 소비 → 가치 실현 → 피드백
 │       │        │         │        │        │         │
 └───────┴────────┴─────────┴────────┴────────┴─────────┘
                    서비스 여정 (Service Journey)
```

## 3. 경험 관리 (Experience Management)

### 3.1 왜 경험 관리가 중요한가?

현대 서비스 관리에서 성과 평가는 더 이상 **SLA 준수율**이나 **가동률**만으로 충분하지 않다. ITIL 5는 **사용자 경험(UX)**, **고객 경험(CX)**, **서비스 일관성**을 서비스 성과의 핵심 지표로 격상시켰다.

| 전통적 지표 (ITIL 4) | 경험 중심 지표 (ITIL 5) |
|---------------------|----------------------|
| 인시던트 해결 시간 (MTTR) | 사용자 만족도 점수 (CSAT) |
| 가동률 (Uptime %) | 순추천지수 (NPS) |
| SLA 준수율 | 서비스 경험 수준 (XLA) |
| 티켓 처리 건수 | 사용자 노력 점수 (CES) |
| 변경 성공률 | 여정 완료율 |

### 3.2 XLA (Experience Level Agreement)

ITIL 5는 전통적인 SLA를 보완하는 **XLA (Experience Level Agreement)** 개념을 강조한다.

| 항목 | SLA | XLA |
|------|-----|-----|
| 측정 대상 | 기술적 성능 (응답 시간, 가동률) | 사용자 경험 (만족도, 편의성, 감정) |
| 관점 | 서비스 제공자 관점 | 서비스 소비자 관점 |
| 목적 | 서비스 수준 보장 | 경험 수준 보장 |
| 데이터 소스 | 모니터링 시스템, 로그 | 설문조사, 피드백, 행동 분석 |
| 활용 | 계약 준수 확인 | 서비스 개선 방향 결정 |

### 3.3 경험 관리 프레임워크

```
┌─────────────────────────────────────────────┐
│              경험 관리 프레임워크              │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │  측정    │→│  분석    │→│  개선    │    │
│  │ Measure  │  │ Analyze │  │ Improve │    │
│  └─────────┘  └─────────┘  └────┬────┘    │
│       ↑                          │         │
│       └──────────────────────────┘         │
│              지속적 개선 루프                │
├─────────────────────────────────────────────┤
│  지표: CSAT, NPS, CES, XLA                  │
│  도구: 설문, A/B 테스트, 행동 분석, AI 분석   │
│  방법: 여정 매핑, 페르소나, 감정 분석          │
└─────────────────────────────────────────────┘
```

## 4. ITIL Experience (Version 5) 도서

ITIL Experience는 ITIL 5의 4대 핵심 도서 중 하나로, **인간 중심 디지털 서비스 설계**를 다룬다.

### 주요 다루는 주제

| 주제 | 설명 |
|------|------|
| 경험 기반 서비스 설계 | 사용자 경험을 중심에 놓는 서비스 설계 방법론 |
| 서비스 여정 매핑 | 사용자가 서비스와 상호작용하는 전체 여정 시각화 |
| XLA 설계와 측정 | Experience Level Agreement 수립 및 모니터링 |
| 감정 분석과 피드백 | 사용자 감정 데이터를 활용한 서비스 개선 |
| 접근성과 포용성 | 다양한 사용자를 위한 포용적 서비스 설계 |
| 개인화 | AI/데이터 기반 개인화된 서비스 경험 제공 |

### 인증 경로에서의 위치

- ITIL Experience는 **Managing Professional (MP)** 경로의 필수 모듈이다
- Foundation 이후 수강 가능
- ITIL Product, ITIL Service와 함께 MP 지정의 3대 핵심 모듈 구성
