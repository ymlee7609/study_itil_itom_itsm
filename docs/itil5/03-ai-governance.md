## 1. AI 거버넌스 개요

**AI 거버넌스**는 ITIL 5에서 가장 주목할 만한 신규 영역이다. AI와 자동화가 IT 환경에 깊이 침투한 현실을 반영하여, AI를 **윤리적(Ethically)**, **안전하게(Securely)**, **전략적(Strategically)**으로 관리하기 위한 공식 프레임워크를 제공한다.

### ITIL 4와의 차이

| 항목 | ITIL 4 | ITIL 5 |
|------|--------|--------|
| AI 다룸 | 자동화의 일부로 간접 언급 | 독립된 거버넌스 프레임워크로 공식화 |
| AI 역량 분류 | 없음 | 6C 모델 (AI Capability Model) 도입 |
| AI 윤리 | 언급 없음 | 윤리적 AI 도입 가이드라인 포함 |
| 인증 모듈 | 없음 | ITIL AI Governance (별도 확장 모듈) |
| AI 용어 | 없음 | 15개 AI 거버넌스 관련 신규 용어 정의 |

---

## 2. AI 거버넌스가 필요한 이유

| 이유 | 설명 |
|------|------|
| 리스크 관리 | AI 시스템의 편향(Bias), 환각(Hallucination), 보안 취약점 관리 필요 |
| 규제 준수 | EU AI Act 등 글로벌 AI 규제에 대한 컴플라이언스 요구 증가 |
| 투명성 | AI 의사결정의 설명 가능성(Explainability) 보장 필요 |
| 책임 소재 | AI가 내린 결정에 대한 책임(Accountability) 체계 수립 필요 |
| 기회 극대화 | 체계적 거버넌스를 통해 AI 도입의 가치를 극대화 |
| 인간-AI 협업 | 인간과 기술의 효과적 협업 모델 구축 |

## 3. 6C 모델 (AI Capability Model)

**6C 모델**은 ITIL 5가 도입한 AI 역량 분류 프레임워크이다. 조직이 AI 활용 사례를 체계적으로 분류하고 관리할 수 있도록 6가지 역량 범주를 정의한다.

### 6가지 역량 (6C)

| # | 역량 | 영문 | 설명 | 실무 예시 |
|---|------|------|------|-----------|
| 1 | **생성** | Creation | 새로운 콘텐츠, 코드, 문서 생성 | 코드 자동 생성, 문서 초안 작성, 디자인 프로토타입 생성 |
| 2 | **큐레이션** | Curation | 기존 데이터의 품질과 관련성 향상 | 데이터 정제, 중복 제거, 지식 베이스 정리, 태깅 자동화 |
| 3 | **명확화** | Clarification | 복잡한 콘텐츠의 이해와 탐색 지원 | 인시던트 티켓 요약, 문서 검색 지원, FAQ 자동 생성 |
| 4 | **인지** | Cognition | 패턴 식별, 숨겨진 인사이트 발견 | 이상 탐지(Anomaly Detection), 예측적 문제 관리, 트렌드 분석 |
| 5 | **소통** | Communication | AI 시스템 내 커뮤니케이션 역량 | 챗봇, 자연어 인터페이스, 다국어 번역, 알림 자동화 |
| 6 | **조율** | Coordination | AI 지원 프로세스 조율 및 자동화 | 워크플로우 자동화, 에스컬레이션 라우팅, 리소스 할당 최적화 |

## 4. AI 도입 가이드라인

ITIL 5는 AI 도입 시 다음 세 가지 단계를 체계적으로 수행할 것을 권고한다.

### 4.1 AI 도입 드라이버 식별

| 드라이버 유형 | 예시 |
|-------------|------|
| 효율성 | 반복 작업 자동화, 처리 시간 단축 |
| 품질 | 일관된 서비스 품질, 인적 오류 감소 |
| 경험 | 사용자 셀프서비스 개선, 개인화된 지원 |
| 혁신 | 새로운 서비스 모델 개발, 예측적 운영 |
| 비용 | 운영 비용 절감, 리소스 최적화 |

### 4.2 리스크 평가

| 리스크 범주 | 고려 사항 |
|-----------|----------|
| 기술적 리스크 | 모델 정확도, 데이터 품질, 시스템 안정성 |
| 윤리적 리스크 | 편향(Bias), 공정성, 프라이버시 |
| 운영적 리스크 | AI 장애 시 폴백(Fallback) 계획, 인간 개입 지점 |
| 규제적 리스크 | 데이터 보호법, AI 규제, 산업별 규제 |
| 조직적 리스크 | 변화 저항, 기술 역량 격차, 문화적 준비도 |

### 4.3 거버넌스 수립

| 거버넌스 요소 | 설명 |
|-------------|------|
| 정책 | AI 사용 정책, 데이터 거버넌스 정책, 윤리 가이드라인 |
| 프로세스 | AI 모델 승인 프로세스, 모니터링/평가 프로세스 |
| 역할/책임 | AI 오너, 데이터 스튜어드, 윤리 위원회 |
| 모니터링 | AI 성능 모니터링, 편향 탐지, 드리프트 감지 |
| 감사 | 정기적 AI 감사, 규제 준수 검증 |

## 5. AI 거버넌스 실무 적용

### 시나리오: IT 서비스 데스크에 AI 챗봇 도입

| 단계 | 활동 | 6C 분류 |
|------|------|---------|
| 1 | 서비스 데스크 티켓 분석 → 자주 묻는 질문 패턴 식별 | Cognition |
| 2 | FAQ 지식 베이스 자동 구축 | Curation |
| 3 | 자연어 기반 챗봇 인터페이스 구현 | Communication |
| 4 | 티켓 자동 분류 및 라우팅 | Coordination |
| 5 | 인시던트 요약 및 해결 가이드 생성 | Clarification |
| 6 | 해결 스크립트 자동 생성 | Creation |
| 7 | 리스크 평가: 잘못된 답변 리스크, 에스컬레이션 규칙 설정 | - |
| 8 | 거버넌스: 인간 검증 단계 추가, 월별 정확도 리뷰 | - |

## 6. Foundation 시험에서의 AI 범위

ITIL 5 Foundation 시험에서 AI 관련 내용은 **인식(Awareness) 수준**으로 다뤄진다.

| 항목 | 설명 |
|------|------|
| 출제 범위 | AI 거버넌스 기본 개념, 6C 모델 용어 |
| 난이도 | 인식 수준 (깊은 기술 지식 불요) |
| 신규 용어 | 15개 AI 거버넌스 관련 신규 용어 |
| 학습 포인트 | AI가 서비스 가치 흐름과 어떻게 통합되는지 이해 |
| 고급 내용 | ITIL AI Governance (확장 모듈)에서 심화 다룸 |

### Foundation에서 알아야 할 AI 핵심 개념

- 6C 모델의 6가지 역량 명칭과 간단한 정의
- AI 거버넌스의 필요성 (리스크, 윤리, 규제)
- 인간과 기술의 협업 원칙
- AI 도입 시 고려해야 할 기본 요소 (드라이버, 리스크, 거버넌스)
